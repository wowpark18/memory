### 🛠️ (3) MVP 핵심 기능 5가지: 프로세스 기반의 순차적 경험 설계
MVP는 다음의 단일 워크플로우를 완벽하게 구현하여, 사용자가 '정보단절'을 느끼고 → 시스템이 '갭(Gap)'을 찾아내어 → 해결책('Intervention')까지 가는 **완전한 서사적 흐름**을 증명하는 데 집중합니다.

1.  **[Input] 다중 출처 비정형 데이터 통합 엔진:**
    *   **기능 스펙:** 텍스트(문서/회의록), 이미지(OCR 스캔된 매뉴얼), 체크리스트 등 이질적인 데이터를 한곳에 업로드할 수 있어야 합니다. (단순 저장 기능이 아닌, 시스템 내에서 **'데이터 연결 가능성'을 자동 진단**해야 함.)
    *   **가치:** "정보의 파편화"라는 초기 Pain Point를 해결하는 첫 관문입니다.

2.  **[Process Mapping] 인터랙티브 프로세스 캔버스 (Flow):**
    *   **기능 스펙:** 사용자가 업로드된 데이터 기반으로 'A $\rightarrow$ B $\rightarrow$ C'와 같은 단계적 워크플로우를 드래그앤드롭 방식으로 시각화합니다. 각 노드는 **필수 전제 조건(Assumption)**을 텍스트 필드로 강제 지정하도록 설계합니다.
    *   **가치:** 단순한 지식 연결이 아닌, *규칙과 순서가 있는 프로세스*를 구축하게 하여 복잡성을 통제 가능하게 만듭니다.

3.  **[Core Logic] Gap & Assumption Detector (AI 엔진):**
    *   **기능 스펙:** 사용자가 정의한 노드(A $\rightarrow$ B) 사이에 **필수 전제 조건이 빠졌거나 논리적 모순**이 발생하는 지점($\text{Gap}$)을 AI가 빨간색으로 하이라이트하고, 그 Gap의 종류를 (정보 부족/논리 오류/자원 부족 등) 카테고리화하여 제시합니다.
    *   **가치:** EPHA의 핵심 가치인 '단절 해소'를 기술적으로 증명하는 부분입니다.

4.  **[Output] 자동 생성 '개입 로드맵(Intervention Roadmap)':**
    *   **기능 스펙:** Gap이 발견될 때마다, 시스템은 해당 Gap을 메우기 위한 **3단계의 구체적인 행동 계획(Action Items)**과 필요한 추가 데이터/지식을 추천합니다. (예: "Gap A 해소 $\rightarrow$ 1) 담당자 인터뷰 필요 / 2) 표준화된 양식 도입 / 3) 관련 규정 문서 연결.")
    *   **가치:** 단순 진단에 그치지 않고, 즉시 실행 가능한 '솔루션 프로세스'를 제공하여 Pay-per-Report의 근거를 마련합니다.

5.  **[UX] 프로세스 마찰 시뮬레이터 (Friction Simulator):**
    *   **기능 스펙:** 사용자가 정의한 프로세스가 실제 운영 환경에서 겪을 수 있는 '가상의 병목 지점(Bottleneck)'과 그로 인해 발생하는 **시간/자원 손실 추정치($\text{ATRI}$)**를 애니메이션으로 시뮬레이션하여 보여줍니다.
    *   **가치:** 감성적인 Pain $\rightarrow$ Relief 스토리텔링을 정량적 데이터와 결합합니다. (기술적 깊이 + 공감 유발)
