### 🎬 [시나리오 1] 데이터 누락으로 인한 KYC 프로세스 실패 (가장 흔한 Gap)
**목표:** 사용자가 필수 정보를 빠뜨리는 가장 기본적인 실수를 시각적으로 경고하고, 어떤 정보가 필요한지 정확히 안내한다.
**핵심 감정 변화:** 혼란 $\rightarrow$ 명확성

| 단계 | 역할/화면 | 대사/UI 지침 (톤앤매너: 차분하고 명료한 시스템 음성) | 기술적 근거 (API 출력 반영) |
| :--- | :--- | :--- | :--- |
| **1. [사용자 입력]** | *전체 화면* / 사용자가 KYC 정보를 입력하는 폼을 작성한다. 이름, 주소 등은 모두 채워졌으나, '최근 거래 활동 증명서' 항목은 비어있다. | *(화면 지침)*: 사용자에게 평이한 배경 음악과 함께 프로세스 진행 바가 녹색으로 움직인다. "KYC 인증을 위해 필요한 정보를 입력해 주세요." | `process_id`: KYC\_Process, `input_metrics`: {Name: 1, Address: 1} (증명서 누락) |
| **2. [시스템 진단]** | *화면 전환* / 사용자가 '인증 완료' 버튼을 클릭하는 순간, 화면 전체가 강렬한 주황색 경고(Warning Red)로 바뀐다. | **[시스템 음성]: "잠시만요. 데이터 불일치 리스크가 감지되었습니다."** (⚠️ 아이콘과 함께 팝업 발생) <br> *사용자 행동:* 당황하여 입력창을 다시 확인한다. | `success`: False, `risk_level`: MEDIUM<br>`message`: 필수 증빙 자료 누락으로 인해 프로세스가 중단됩니다. |
| **3. [Gap 분석 및 Edge 제시]** | *팝업 상세* / 팝업이 열리며 '문제점'과 '해결 과정'의 구분이 명확하게 나타난다. (Pain $\rightarrow$ Gap $\rightarrow$ Edge 구조) | **[시스템 음성]: "분석 결과, '거래 증명서(Transaction Proof)' 정보가 누락되었습니다."**<br>**[Gap 상세]:** 현재 단계에서는 *최근 3개월간의 거래 내역*이 반드시 필요합니다. (⚠️ 빨간색으로 강조)<br>**[Edge 안내]:** [해결책] 해당 문서를 업로드해 주시거나, 관리자에게 연락하여 수동 인증을 요청해주세요. | `identified_risks`: [{metric: 'Transaction Proof', status: 'Missing'}], <br>`recommended_action`: {Action: 'Upload Document', Guide: '3개월 이내 거래 내역서 제출'} |

### 🎬 [시나리오 2] 비표준 프로세스 사용으로 인한 금융 리스크 (중급 Gap)
**목표:** 규정된 흐름(Standard Flow)을 건너뛰고 임의로 데이터를 처리할 때 발생하는 잠재적 위험성을 보여주어, 정해진 절차를 따르게 유도한다.
**핵심 감정 변화:** 안도감 $\rightarrow$ 위협 $\rightarrow$ 체계화

| 단계 | 역할/화면 | 대사/UI 지침 (톤앤매너: 전문가가 옆에서 조용히 코칭하는 느낌) | 기술적 근거 (API 출력 반영) |
| :--- | :--- | :--- | :--- |
| **1. [사용자 입력]** | *프로세스 맵* / 사용자가 A $\rightarrow$ B $\rightarrow$ C 순서로 진행해야 할 프로세스를, 임의로 A와 C를 건너뛰고 D(다른 경로)부터 시작하려 한다. | *(화면 지침)*: 시스템이 흐름도를 보여주며 'A'에서 출발하여 다음 단계가 무엇인지 시각적으로 가이드한다. "다음 단계를 선택해 주세요." | `process_id`: Risk\_Flow\_Check, `input_metrics`: {D_start: 1} (잘못된 시작점) |
| **2. [시스템 진단]** | *경고 발생* / 사용자가 D를 선택하자마자, 흐름도가 갑자기 끊기며 경고 메시지가 나타난다. 배경색이 어둡고 불안정한 청록색(Anxiety Blue)을 띠게 한다. | **[시스템 음성]: "잠시만요. 현재의 프로세스 흐름은 표준 규정에 위배됩니다."**<br> *사용자 행동:* 혼란스러워하며 'A'를 다시 클릭한다. | `success`: False, `risk_level`: HIGH <br>`message`: 정의된 핵심 연결 과정(Edge)을 벗어났습니다. 재검토가 필요합니다. |
| **3. [Gap 분석 및 Edge 제시]** | *상세 분석* / 시스템이 문제의 '흐름' 자체를 붉은색으로 표시하고, 올바른 경로를 녹색 화살표로 보여준다. | **[시스템 음성]: "위험 경고: A $\rightarrow$ B 순서가 핵심 프로세스입니다."**<br>**[Gap 상세]:** 현재 사용하신 'D'는 중간 단계의 예외 처리 과정만 가능하며, 초기 검증(A)을 생략할 수 없습니다. (❌ 표시)<br>**[Edge 안내]:** **[필수 경로]** A $\rightarrow$ B $\rightarrow$ C 순서로 단계를 진행해 주십시오. 이 흐름이 프로세스의 무결성을 보장합니다. | `identified_risks`: [{metric: 'Flow Integrity', status: 'Broken'}], <br>`recommended_action`: {Action: 'Revert Flow', Guide: '프로세스 A를 먼저 재검토해주세요.'} |

### 🎬 [시나리오 3] 윤리적 책임감 결여로 인한 정보 단절 (가장 심화된 Gap)
**목표:** 단순히 기술적인 오류가 아닌, *윤리적/규제적 의사결정 과정* 자체에 문제가 있음을 경고하며, 기업의 가치(Ethical Governance)를 전달한다.
**핵심 감정 변화:** 무지 $\rightarrow$ 죄책감 $\rightarrow$ 책임감

| 단계 | 역할/화면 | 대사/UI 지침 (톤앤매너: 가장 권위적이고 깊이 있는 목소리. 신뢰감을 강조.) | 기술적 근거 (API 출력 반영) |
| :--- | :--- | :--- | :--- |
| **1. [사용자 입력]** | *의사결정 대시보드* / 사용자가 A 데이터를 기반으로 빠르게 결론을 내리려 한다. 시스템은 '규제 가이드라인 V2.1'이 있음에도 불구하고, 과거 버전(V1.0)에 근거하여 최종 보고서를 작성한다. | *(화면 지침)*: 사용자는 자신감 있게 다음 단계를 진행하며, 모든 것이 순조로운 것처럼 보인다. "최종 분석 완료." | `process_id`: Governance\_Report, `input_metrics`: {Data\_v1.0: 1} (구버전 데이터 활용) |
| **2. [시스템 진단]** | *강렬한 경고* / 화면이 가장 어두운 남색(Deep Warning Blue)으로 변한다. 일반적인 '에러'가 아닌, '경고(Warning)' 메시지가 뜬다. | **[시스템 음성]: "중대한 윤리적/규제적 불일치가 발견되었습니다."** (🚨 경고 사이렌 소리가 짧게 울림) <br> *사용자 행동:* 무엇이 문제인지 찾아보려 한다. | `success`: False, `risk_level`: CRITICAL<br>`message`: 사용된 기준(Standard)의 시점과 법적 근거가 최신화되지 않았습니다. |
| **3. [Gap 분석 및 Edge 제시]** | *최상위 보고서* / 시스템이 '규제 변화 흐름' 그래프를 보여준다. (시간적 맥락/Temporal Context 강조) | **[시스템 음성]: "경고: 이 결정은 최신 법률 기준(V2.1)을 충족하지 못합니다."**<br>**[Gap 상세]:** 사용하신 V1.0 기준은 6개월 전의 규정이며, 현재는 '데이터 주권' 및 'Ethical Flow'가 핵심입니다. (💔 빨간색으로 강조)<br>**[Edge 안내]:** **[필수 조치]** 최신화된 가이드라인(V2.1)을 기준으로 재검토해야 합니다. 저희 시스템은 시간의 흐름에 따른 규제 변화를 추적하여, 이 정보 단절을 해소해 드립니다. | `identified_risks`: [{metric: 'Compliance', status: 'Outdated'}], <br>`recommended_action`: {Action: 'Update Standard', Guide: '규제 가이드라인 V2.1 재적용 및 전면 검토 필요.'} |
