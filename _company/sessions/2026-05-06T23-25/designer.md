# 🚨 'Gap Alert' 컴포넌트 인터랙션 상세 스펙 (Developer Spec)

본 문서는 EPHA의 핵심 기능인 정보 단절 지점($F_{Gap}$)을 재무적 리스크(L-Cost)로 정량화하여 사용자에게 경고하는 'Gap Alert' 컴포넌트를 개발자가 바로 코딩할 수 있도록 3단계 상태별 마이크로 인터랙션 가이드라인으로 작성되었습니다.

---

## ⚙️ 기본 설정 및 전역 변수 (Global Variables)

| 항목 | 값/설명 | 용도 |
| :--- | :--- | :--- |
| **Alert Color** | `#D92A3B` (High-Risk Red) / `rgba(217, 42, 59, X)` | 경고 발생 시의 주 색상. 강렬한 위험 신호. |
| **Process Color** | `#3498DB` (Deep Blue) / `rgba(52, 152, 219, Y)` | 데이터 로딩 및 계산 과정의 안내색. 신뢰감 구축. |
| **Success Color** | `#2ECC71` (Safe Green) | 해결책 제시 후의 안도/통찰 색상. |
| **Transition Time** | `300ms ~ 600ms` | 모든 상태 전환에 적용될 애니메이션 시간 범위. |

---

## 🚀 [Stage 1] Gap Alert 발생 (Trigger State)

사용자가 명확한 정보 단절 지점($F_{Gap}$)을 발견했을 때, 시스템이 능동적으로 개입하여 경고를 발생시키는 단계입니다. **목표: 즉각적 주의 집중 및 문제 인식 유도.**

| 요소 | 액션/트리거 | 애니메이션/UX 스펙 |
| :--- | :--- | :--- |
| **시각적 트리거** | 시스템이 $F_{Gap}$을 감지하는 순간. (예: 사용자가 핵심 키워드 A와 관련 데이터를 찾아야 할 때, 연결된 B 데이터가 부재함) | 1. **배경 플래시:** 화면 중앙에 `#D92A3B` 색상의 빛 깜빡임(Flash/Pulse) 효과를 200ms 동안 적용. <br> 2. **경계선 활성화:** 현재 보고 있는 정보 영역 주변의 테두리가 `굵기 2px`, `색상 #D92A3B`로 즉시 발광하며 깜빡임(Pulsing Glow). |
| **알림 모달** | 'Gap Alert: 필수 데이터 연결 실패' 메시지와 함께 오버레이. | **진입 애니메이션:** 배경으로부터 빠르게 확대되며 나타남 (Scale-up from 0.9 to 1.0, Duration: 250ms). <br> **헤드라인 타이포그래피:** `Bold`, `#D92A3B`로 강조 처리. |
| **오디오/햅틱** | (모바일 환경) 날카롭고 짧은 경고음(Alert Tone) 재생 및 진동 피드백 발생. | - |

---

## 🔄 [Stage 2] L-Cost 계산 프로세스 (Processing State)

시스템이 $F_{Gap}$을 재무적 리스크($L$-Cost)로 정량화하는 과정입니다. 사용자에게 '계산 중'이라는 느낌보다, **'지능적으로 분석하고 있다'**는 신뢰를 주어야 합니다. **목표: 지연 시간을 가치 있는 경험으로 전환.**

| 요소 | 액션/트리거 | 애니메이션/UX 스펙 |
| :--- | :--- | :--- |
| **화면 전환 (Overhaul)** | Stage 1 모달이 사라지며, 전용 로딩 컴포넌트로 대체. | `Transition Time: 400ms`. 배경은 약간 어둡고(Opacity 0.9), 분석적인 그리드 패턴이 은은하게 오버레이됨. |
| **프로세스 시각화** | L-Cost 공식($L$-Cost = Loss Frequency $\times$ Impact Scale)의 각 변수가 실시간으로 계산되는 과정을 애니메이션으로 표현. | 1. **노드 활성화:** 관련 데이터 노드가 배경에서 무작위로 발견되며(`#3498DB` 점멸), '데이터 수집 중...', '패턴 분석 시작...' 등의 문구가 순차적으로 타이핑되듯 표시됨. <br> 2. **엣지 생성:** 발견된 노드들이 서로 연결되는 선(Edge)이 빛나는 트레일(Glowing Trail) 애니메이션으로 그려짐. (지식 그래프 시각화 활용). |
| **L-Cost 카운터** | 최종 L-Cost가 계산됨에 따라 실시간 숫자가 증가하는 '카운트업' 효과 구현. | **애니메이션:** 0에서 시작하여 목표 값(예: $1,250,000$)까지 부드럽고 빠르게 (Easing Curve 적용) 수치가 올라감. <br> **피드백:** 카운터 주변에 `#3498DB` 색상의 에너지 필드가 감도는 듯한 미세한 애니메이션 추가. |
| **사용자 피드백** | 로딩 바 대신, 계산 단계별로 진행도를 표시하는 '프로세스 막대' 사용. (예: [데이터 수집] $\rightarrow$ [패턴 매핑] $\rightarrow$ [리스크 정량화]) | 각 섹션 완료 시 짧은 성공음과 함께 `Scale-up` 효과를 주어 단계별 성취감을 부여. |

---

## ✅ [Stage 3] L-Cost 리스크 진단 (Resolution State)

계산이 완료되어 최종 $L$-Cost가 확정되고, 사용자에게 구체적인 해결책을 제시하는 단계입니다. **목표: 통찰력 제공 및 다음 액션 유도.**

| 요소 | 액션/트리거 | 애니메이션/UX 스펙 |
| :--- | :--- | :--- |
| **최종 결과 표시** | 계산 과정의 오버레이가 사라지고, 깨끗하고 밝은 (하지만 여전히 전문적인) UI로 전환. | `Transition Time: 500ms`. 배경에 흐르던 그리드 애니메이션이 서서히 사라지며 안정된 상태로 복귀. |
| **L-Cost 명시** | 최종 L-Cost 값을 가장 크게, 중앙에 배치합니다. | **애니메이션:** `#D92A3B` 경고색에서 점차 `#2ECC71` 안심색으로 색상이 변하며 (Color Shift), 수치가 팝업되며 등장. `Font Size: Huge`, `Bold`. |
| **리스크 구조화** | L-Cost가 왜 발생했는지(원인)와 어떻게 해결할지(해결책)를 명확히 분리하여 제시합니다. | 1. **[Gap] 섹션:** 원인이 된 $F_{Gap}$ 노드를 빨간색으로 유지하며 간략 설명. <br> 2. **[Solution] 섹션:** EPHA 서비스의 작동 과정을 `#3498DB` 색상의 깔끔한 플로우 차트로 시각화하여, '이것을 보강하면 된다'는 명확한 방향성을 제시함. |
| **CTA (Call To Action)** | 다음 단계로 이동할 버튼 (`Mitigate Gap`, `View Detailed Report`) 활성화. | 클릭 가능 상태가 될 때만(`#3498DB` Hover Effect) 입체적인 그림자(Shadow)와 옅은 `:active` 상태 변화를 주어 상호작용을 유도함. |
