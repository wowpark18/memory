### 📊 [Blind Spot] 간과하기 쉬운 측정 가능한 재무적 리스크 3가지

| 구분 | 리스크 이름 (Blind Spot) | 정의 및 문제점 | 데이터 포인트 (측정 가능 요소) | $L_{Gap}$ 정의 (현재의 부족함) |
| :---: | :--- | :--- | :--- | :--- |
| **1** | **시맨틱 드리프트 리스크** *(Semantic Drift Risk)* | AI 모델이나 시스템이 학습한 '지식의 의미'가 시간이 지나면서 실제 비즈니스 환경의 변화(용어, 맥락)를 따라가지 못하여 발생하는 오작동 및 판단 오류. (데이터는 존재하나, 해석 체계가 노후화됨) | 1. **도메인 용어 불일치 지수:** 산업 표준 용어 대비 내부 시스템 사용 빈도의 편차 비율. <br>2. **의미론적 거리(Semantic Distance):** 최신 트렌드 키워드와 과거 데이터셋 간의 평균 임베딩 벡터 거리 값 (Cosine Similarity 활용). | *지식의 해석 유효성.* 데이터를 단순히 모으는 것($L_{Data}$)을 넘어, 해당 데이터가 현재 비즈니스 맥락에서 어떤 의미를 가지는지(Contextual Validity) 판단하는 **실시간 '의미론적 검증 시스템'**이 부재함. |
| **2** | **디지털 종속성 리스크** *(Digital Dependency Risk)* | 핵심 서비스나 운영 프로세스가 특정 외부 API, 플랫폼, 또는 단일 공급자(Single Point of Failure)에 과도하게 의존하여 발생하는 기능 정지 및 비용 폭등 위험. (공급망 불확실성의 디지털화된 형태) | 1. **핵심 API 의존 계수:** 매출액 대비 상위 3개 외부 API 제공업체 지출 비중 및 단일 실패 시 예상 손실 금액 ($L_{Failure}$). <br>2. **서비스 복원 시간(RTO):** 주요 서비스 장애 발생 시, 대체 시스템 구축에 필요한 평균 시간 (시간/비용). | *시스템의 회복 탄력성.* 외부 의존성을 사후적으로 파악하는 것을 넘어, 각 컴포넌트가 어떤 대체 경로를 가질 수 있는지(Alternative Path Mapping)를 **설계 단계에서부터 예측하고 시뮬레이션**할 수 있는 아키텍처 분석 시스템이 부족함. |
| **3** | **조직 지식 단편화 리스크** *(Knowledge Fragmentation Risk)* | 핵심 인재의 퇴사, 부서 간 사일로(Silo) 현상 등으로 인해 특정 프로젝트나 프로세스에 필요한 지식이 여러 비정형화된 저장소(개인 메모, 메신저 대화, 오래된 문서 등)에 파편화되어 관리되는 위험. (가장 감지하기 어려우며 재무적 손실이 크다.) | 1. **정보 접근성 비용:** 특정 업무 관련 지식을 찾기 위해 평균적으로 소요되는 직원 시간(Man-hour/Quarter). <br>2. **핵심 인재 퇴사 예측 리스크 점수:** 해당 인물이 보유한 고유 지식 자산의 가중치와 대체 난이도를 종합하여 산출한 임계값 (위험도 0~100점). | *지식의 구조적 연결성.* 단순히 문서화된 정보를 모으는 것($L_{Archive}$)을 넘어, 파편화된 지식 조각들을 **어떤 '관계(Relationship)'로 연결해야 재활용 가능한 시스템 자산**으로 만들 수 있는지(Knowledge Graph Mapping)를 동적으로 추론하는 메커니즘이 부재함. |
