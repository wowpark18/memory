## 🛠️ Mock-up Data Source: $L_{Total}$ 시뮬레이션 데이터셋 설계 및 생성

### 🎯 목표 정의
이 가상 데이터 소스는 '정보 단절(Information Silo)'과 '프로세스 비효율성'으로 인해 발생하는 **측정 가능한 잠재적 손실($L_{Total}$)을 역추론**할 수 있도록 구조화되었습니다. 시나리오의 핵심은 'A 부서가 보유한 중요한 정보(Know-how)를 B 시스템이 인지하지 못해 발생한 기회비용'입니다.

### 💡 가상 시나리오 배경
*   **회사:** EPHA (에파)
*   **프로젝트:** 차세대 리스크 예측 모듈 개발 (AI 기반).
*   **문제 상황:** 프로젝트 팀이 A(법무팀), B(개발팀), C(마케팅팀) 세 부서의 지식을 통합하여 새로운 기능을 만들려 했으나, 각 부서가 사용하는 데이터 표준과 용어 체계가 달라 핵심 리스크 요소를 놓치고 있습니다.

### 💾 Mock Data Structure (JSON/YAML 스타일 정의)
모든 데이터는 **[지식 노드 ID]**를 통해 연결되며, 이 연결 과정에 손실이 발생합니다.

| 필드명 | 타입 | 설명 | 예시 값 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| `Knowledge_ID` | String | 고유 지식 노드 ID (Source-Domain 조합) | LAW-001\_IP, DEV-015\_API | 필수 식별자 |
| `Domain` | Enum | 정보의 출처/영역 | Legal / Development / Marketing | 데이터 분류 |
| `Core_Data_Point` | String | 해당 노드가 가진 핵심 사실 또는 지식 파편 | "특정 기술 스택은 2025년까지 라이선스 만료 예정." | 실제 값 (사실) |
| `Metadata_Tag` | Array | 관련 개념어, 키워드 | ["IP", "라이선스", "기술 부채"] | 연결성 강화용 태그 |
| `Source_Owner` | String | 정보의 원 소유자 (실제 담당자/팀) | 법무팀 김대리 | 책임 소재 명시 |

### 📉 $L_{Total}$ 발생 시뮬레이션 테이블 (The Gap Analysis)
다음은 Mock Data를 사용하여 손실을 추론하는 핵심 로직입니다. 개발자는 이 **'Connection Rule'**에 실패했을 때의 비용($L$)을 계산해야 합니다.

| 연결 규칙 (`Rule`) | 필요 노드 A (`ID`, `Domain`) | 필요 노드 B (`ID`, `Domain`) | 필수 변수/논리적 관계 | Gap 상태 (실제) | 잠재 손실 ($L_{Total}$) 추정 근거 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **R1: 기술-법률 리스크** | `LAW-001_IP` (법무) | `DEV-015_API` (개발) | "특정 라이선스 만료 여부" $\rightarrow$ "사용 불가 API 사용 위험성" | ❌ B 부서가 A의 '만료일'을 인식하지 못함. | **[A] 법적 위반 리스크:** API 사용 중단에 따른 서비스 다운타임 비용 (예상 매출 손실 1억 원). |
| **R2: 시장-기술 기회** | `MKT-003_Trend` (마케팅) | `DEV-015_API` (개발) | "시장 트렌드 변화" $\rightarrow$ "필수적으로 구현해야 할 신규 기능 정의" | ❌ C 부서가 '경쟁사 분석 데이터'를 개발팀에 전달하지 않음. | **[B] 기회비용 손실:** 시장 선점 실패로 인한 추가 매출 발생분 (예상 이익 손실 5천만 원). |
| **R3: 프로세스 비효율성** | `DEV-022_Workflow` (개발) | `LAW-010_Compliance` (법무) | "업데이트된 컴플라이언스 요건" $\rightarrow$ "필수 수정 로직/테스트 케이스 추가" | ⚠️ A 부서가 최신 가이드라인을 개발팀에 *부분적*으로 전달함. | **[C] 프로세스 지연 비용:** 불완전한 요구사항 정의로 인한 재작업 시간 및 인력 투입 손실 (추정 공수: 30인일). |
