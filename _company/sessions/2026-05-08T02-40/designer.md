## EPHA C-Level 발표용 디자인 브리프: $L_{Cost}$ 동적 서사 구조화
본 브리프는 Business가 확정한 '위험 경고 $\rightarrow$ 분석 $\rightarrow$ 해결책 제시'의 3단계 스토리라인을 기반으로, 단순한 정보 전달이 아닌 **경험(Experience)**과 **긴장감(Tension)**을 디자인 요소로 활용하는 C-Level 프레젠테이션 템플릿 스펙을 정의합니다.

---

### I. [전체 브랜드 가이드] EPHA Deck Standard

| 항목 | 규정/설명 | 구체적 스펙 (Hex Code / Font) |
| :--- | :--- | :--- |
| **톤앤매너** | 딥테크, 권위적(Authoritative), 미니멀리즘. 정보의 '단절'과 '연결'에 집중. | - |
| **컬러 팔레트** | **Background:** 깊은 밤하늘 같은 블랙/다크 네이비 (신뢰감). <br>**Primary Text:** 화이트 또는 라이트 그레이 (#F0F0F0) <br>**Accent 1 (위험):** 시각적 충격의 Red Pulse. #D94A3E (진동하는 느낌 부여) <br>**Accent 2 (해결/통찰):** 발견과 연결을 상징하는 Electric Cyan Blue. #00C8FF (희망, 지능적 연결) | **폰트:** Inter 또는 Pretendard (Sans-serif 계열). 가독성이 높고 기술적인 느낌 유지. |
| **레이아웃 원칙** | 그리드 시스템 기반의 대칭 구조를 기본으로 하되, 데이터가 흐르는 방향(Flow)을 강하게 강조해야 함. 여백은 핵심 메시지를 부각하는 '침묵'처럼 활용. | - |

### II. [핵심 컴포넌트 스펙] Red Pulse 경고 시스템 (The Critical Component)

$L_{Cost}$는 텍스트가 아닌, *시각적 고장*으로 표현되어야 합니다. 이 컴포넌트는 단순한 빨간색 박스가 아니라 **상태 변화에 따라 애니메이션이 적용되는 인터랙티브 요소**로 정의합니다.

| 상태 (State) | 시각 효과/애니메이션 스펙 | 재무적 의미 (메시지) |
| :--- | :--- | :--- |
| **1. Normal (대기)** | 투명도 0%의 미세한 그리드 패턴이 배경에 깔려 있음. 안정적이고 차분함. | 현재 프로세스는 정상적으로 작동하고 있습니다. |
| **2. Warning (경고 감지)** | 배경 그리드의 특정 영역(예: 데이터 전송 구간)에서 **Red Pulse**가 낮은 주파수로 깜빡이며 확장됨 (Pulse 1-3초). 색상은 #D94A3E로 설정하며, 그라데이션이 아닌 *디지털 노이즈*처럼 표현되어야 함. | 정보의 단절(Information Gap) 발생 가능성이 포착되었습니다. 주의 필요. ($L_{Info}$ 위험 신호) |
| **3. Critical (위험 확정)** | Red Pulse가 멈추지 않고 빠르게 진동하며, 해당 영역 전체를 붉은색 오버레이가 감싸는 듯한 애니메이션(Flash/Flicker). 강렬하고 불안정한 느낌을 주어야 함. 배경 그리드 패턴 자체가 왜곡되는 시각 효과 추가 필수. | **심각한 재무적 손실($L_{Cost}$) 발생 중.** 즉각적인 개입이 필요합니다. (가장 큰 충격 지점) |
| **4. Resolved (해결)** | Critical 상태를 거쳐, Cyan Blue의 부드러운 파동(Wave) 애니메이션이 붉은색 영역을 점진적으로 덮으면서 사라짐. 배경 그리드가 다시 안정적인 패턴으로 돌아옴. | EPHA가 개입하여 손실 경로가 분석되고 해결책이 제시되었습니다. ($ROI$ 증명 지점) |

### III. [페이지별 디자인 스펙] (총 3가지 템플릿)

#### 📄 Template A: Hook - 위기 고조 및 문제 제기 페이지
*(Business의 'Hook' 단계 대응)*

*   **목표:** 청중에게 공포와 궁금증(Fear & Curiosity)을 극대화하여 주의를 집중시킨다.
*   **레이아웃 스펙:** 비대칭적, 중앙에 거대한 *손실 발생 영역* 배치.
*   **핵심 요소:**
    1.  **헤드라인:** 전체 슬라이드의 60% 이상을 차지할 만큼 크고 충격적인 문구 ("당신의 프로세스는 현재 $X$만큼의 돈을 잃고 있습니다.").
    2.  **시각화 (필수):** 좌측에 '현재 시스템 흐름도'를 배치하고, 핵심 데이터 전이 구간에 **Red Pulse(Critical State)** 컴포넌트를 적용하여 강하게 비활성화 상태를 보여준다.
    3.  **텍스트:** 최소한의 서술만 사용. 숫자와 비율($L_{Total}$)을 가장 큰 폰트로 강조하며, 나머지 설명은 작은 글씨로 '질문' 형태(예: "왜? 어디서?")로 배치하여 궁금증 유발.

#### 📄 Template B: Diagnosis - 손실 원인 분석 페이지
*(Business의 'Diagnosis' 단계 대응)*

*   **목표:** 복잡한 개념을 구조화하고, 문제의 근원을 논리적으로 분해하여 신뢰도를 확보한다.
*   **레이아웃 스펙:** 3분할 (Rule of Three) 그리드 시스템 채택. 전문적이고 분석적인 느낌 강조.
*   **핵심 요소:**
    1.  **제목/서브헤드:** "$L_{Total}$은 단일 문제가 아닙니다. 정보와 과정의 분리입니다." 와 같이, 복합성을 정의하는 문구 사용.
    2.  **시각화 (필수):** 3개의 독립적인 모듈 박스($L_{Info}$, $L_{Process}$, $L_{Total}$)를 배치한다. 각 박스는 **Gap 그래프 또는 Venn Diagram 형태의 시뮬레이션**을 통해 개념적 연관성(교집합)과 손실 크기(개별 영역)를 동시에 보여줘야 한다.
    3.  **데이터:** 숫자를 중심으로 구성 (예: "정보 단절로 인한 시간 지연률: 45%"). Cyan Blue의 강조색을 사용하여 '분석된 수치'임을 명확히 한다.

#### 📄 Template C: Solution - 해결책 및 ROI 제시 페이지
*(Business의 'Solution' 단계 대응)*

*   **목표:** 문제를 극적으로 해소하고, EPHA가 유일한 답안(The Optimal Path)임을 자신감 있게 전달한다.
*   **레이아웃 스펙:** 상향 곡선 (Upward Trajectory). 명료함과 역동성을 동시에 표현해야 한다.
*   **핵심 요소:**
    1.  **시각화 (필수):** 좌측에 **'기존 프로세스(Pain)'를 나타내는 붉은색의 불안정한 흐름도**를 배치하고, 우측에 EPHA 도입 후 **'개선된 프로세스(Relief)'를 나타내는 매끄러운 Cyan Blue의 곡선형 플로우차트**를 대비시킨다.
    2.  **ROI 공식:** $ROI$ 공식을 슬라이드 중앙 하단에 배치하고, 이 수식이 단순 계산이 아닌 '회복되는 가치'임을 강조하기 위해 화살표와 연결선을 사용한다.
    3.  **CTA/마무리:** 다음 단계(Next Steps)를 1~2개의 명확한 액션 아이템으로 요약하여 제시하며 마무리한다.
