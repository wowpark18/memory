### 🏥 Case 1: 의료 및 헬스케어 산업
**리스크 유형:** 정보 단절로 인한 환자 데이터 비호환성 및 법규 위반 (HIPAA/GDPR 준수)
**Pain Point:** 다기관에서 수집된 전자의무기록(EHR)과 유전체 데이터를 개별 시스템이 보유하여, 특정 질병의 진단에 필요한 '통합적 환자 프로세스'를 구성할 수 없음. (정보 단절 해소 실패)

| 구분 | 내용 상세 기술 |
| :--- | :--- |
| **(1) 구체적 문제 상황** | 여러 병원이나 연구 기관에서 파편화된 진단 기록, 처방전, 유전자 분석 데이터가 각기 다른 포맷과 시스템에 저장되어 있어, 의료진이 환자 전체의 '정보 흐름'을 한눈에 파악할 수 없어 오진 또는 치료 지연 발생. |
| **(2) 정량적 손실 규모 (Stakes)** | **법적/금전적 손실:** GDPR 위반 시 전 세계 매출액의 최대 4%까지 벌금이 부과될 수 있음. 또한, 데이터 비호환성으로 인한 의료 과실 소송 건당 평균 배상액은 최소 수억 원 이상 예상됨. |
| **(3) EPHA 프로세스 해결책** | EPHA는 개별 데이터를 단순히 저장하는 것이 아니라, **'진단 의사결정 경로(Diagnostic Process Flow)'** 자체를 추적합니다. A기관의 기록과 B기관의 유전체 데이터 사이에 필요한 '임상 가이드라인 검토 과정'이라는 Gap을 식별하고, 이를 메울 수 있는 표준화된 프로세스 연결 노드 및 근거 문서를 제시하여 오진 리스크를 최소화합니다. |

### 💰 Case 2: 금융 및 핀테크 산업
**리스크 유형:** 국제 자금 세탁 방지(AML) 규제 준수 실패
**Pain Point:** 국경 간 거래나 복잡한 기업 구조에서 관련된 모든 법적 주체와 자금의 흐름을 실시간으로 연결하지 못하여, 의심스러운 거래 패턴을 감지하는 데 실패함.

| 구분 | 내용 상세 기술 |
| :--- | :--- |
| **(1) 구체적 문제 상황** | A국가 기업이 B국가의 법인 계정을 통해 자금을 이동시키고, 최종적으로 C국의 개인에게 전달되는 복잡한 구조에서 '자금의 출처 및 흐름'을 단일 프로세스로 추적하지 못함. KYC/AML 규정 위반 위험 노출. |
| **(2) 정량적 손실 규모 (Stakes)** | **법적/금전적 손실:** 금융기관은 AML 실패 시, 관련 법규에 따라 막대한 벌금(수백만 달러 단위)과 함께 영업 중단 명령 및 신뢰도 하락으로 인한 거래처 상실을 겪음. 미준수로 인해 발생하는 시간당 운영 지연 비용 역시 막대함. |
| **(3) EPHA 프로세스 해결책** | EPHA는 모든 금융 주체와 트랜잭션을 단순히 노드-엣지로 연결하는 것이 아니라, **'자금 흐름의 합법성 검증 프로세스'**를 모델링합니다. 각 국적별 자금세탁 방지 규정(Law A $\rightarrow$ Law B)을 비교 분석하여, 어느 단계에서 법규 Gap이 발생하는지 시각화하고, 해당 Gap을 메우기 위한 필수 서류/검토 과정을 자동 추천합니다. |

### 🚧 Case 3: 건설 및 엔지니어링 산업
**리스크 유형:** 프로젝트 설계 기준과 현장 데이터 간의 불일치 (Code Compliance)
**Pain Point:** 프로젝트가 진행되는 과정에서 건축법, 안전 규정, 사용 자재 표준 등이 지속적으로 업데이트되는데, 초기 설계 단계에 고정된 지식(Blueprint)이 최신 법규를 반영하지 못하여 공사 중단 리스크 발생.

| 구분 | 내용 상세 기술 |
| :--- | :--- |
| **(1) 구체적 문제 상황** | 5년 전 설계된 건축 도면 A와 현재 적용되는 소방/내진 기준 B가 상충하는 지점 발생. 이로 인해 특정 구조물에 대한 안전성 검토 프로세스 자체가 무효화되어 공사 진행이 불가해짐. |
| **(2) 정량적 손실 규모 (Stakes)** | **법적/금전적 손실:** 규정 미준수로 인한 프로젝트 일시 중단 비용은 하루만 해도 수억 원에 달하며, 구조 안전 문제 발생 시 인명 피해 및 막대한 민사상 배상 책임이 따른다. |
| **(3) EPHA 프로세스 해결책** | EPHA는 설계 도면을 단순 데이터로 취급하지 않고, **'최신 규정 준수 검증 프로세스'**를 구축합니다. 초기 모델링 단계에서 '현재 법규(Context Node B)'가 '기존 설계 구조(Context Node A)'에 미치는 영향을 시뮬레이션하고, 어떤 요소(Gap)가 변경되어야 하는지 정량적 리스크 지표와 함께 제시하여 재설계 과정을 최소화합니다. |

### 🌐 Case 4: 데이터 프라이버시 및 마케팅 산업
**리스크 유형:** 국가별/플랫폼별 사용자 동의 처리 불일치 (Consent Management)
**Pain Point:** 글로벌 서비스를 운영하며, 유럽(GDPR), 미국(CCPA), 아시아 등 각기 다른 지역의 개인정보 수집 및 활용 동의 절차를 일관성 있게 관리하지 못해 법적 제재에 노출됨.

| 구분 | 내용 상세 기술 |
| :--- | :--- |
| **(1) 구체적 문제 상황** | 한국 사용자에게는 '마케팅 이용 동의'를 받고, EU 사용자에게는 별도의 '명시적인 목적 기반 동의'가 필요한데, 시스템이 이를 하나의 프로세스로 처리하여 GDPR 위반 위험을 초래함. (다중 규제 환경에서의 정보 단절) |
| **(2) 정량적 손실 규모 (Stakes)** | **법적/금전적 손실:** 전 세계 데이터 프라이버시 법규 미준수 시, 벌금은 매출액 대비 높은 비율로 책정되며, 브랜드 신뢰도 하락으로 인한 장기적인 고객 이탈(LTV 감소)이 발생함. |
| **(3) EPHA 프로세스 해결책** | EPHA는 사용자의 데이터를 단순 수집하는 것을 넘어, **'사용자 동의 및 권리 보호 프로세스 지도(Consent Process Map)'**를 작성합니다. 데이터가 어느 국가에서 어떤 목적과 범위로 활용되어야 하는지 규제 지도를 매핑하고, '이 프로세스를 완료하려면 (나라별) 이 조건이 필요하다'는 구체적인 가이드라인을 실시간으로 제시하여 법적 준수 여부를 사전에 검증합니다. |

### ⛓️ Case 5: 공급망 및 ESG(환경·사회·지배구조) 산업
**리스크 유형:** 원자재의 출처 불투명성 및 윤리적 책임 추적 실패 (Supply Chain Transparency)
**Pain Point:** 특정 부품이나 원자재가 분쟁 지역에서 유래했거나, 노동 착취 등 비윤리적인 과정을 거쳤을 가능성을 공급망 전반에 걸쳐 완벽하게 추적하지 못하여 브랜드 리스크 및 시장 퇴출 위험에 처함.

| 구분 | 내용 상세 기술 |
| :--- | :--- |
| **(1) 구체적 문제 상황** | 최종 제품의 핵심 부품 A가 여러 공급업체를 거치며 복잡한 경로를 이동했지만, 각 단계마다 노동 환경 감찰 보고서나 원산지 증명서 등 필수적인 '윤리적 검증 프로세스' 기록이 누락됨. (공급망 정보 단절) |
| **(2) 정량적 손실 규모 (Stakes)** | **법적/금전적 손실:** ESG 기준 미달로 인한 투자 철회, 주요 시장에서의 불매 운동 발생 시 브랜드 가치 하락은 천문학적이며, 특정 원자재의 경우 수출입 금지 조치가 내려질 수 있음. |
| **(3) EPHA 프로세스 해결책** | EPHA는 모든 공급망 참여 주체와 자재 이동에 대해 **'윤리 및 규제 검증 프로세스 흐름도'**를 생성합니다. 원자재의 출발점부터 최종 소비 단계까지 거쳐야 할 필수 감찰/검증 단계를 정의하고, 각 단계에서 발생하는 '정보 공백(Gap)'을 찾아내어 이를 메울 수 있는 제3의 감사 기관 연결이나 추가 서류 제출 과정을 추천하여 투명성을 확보합니다. |
