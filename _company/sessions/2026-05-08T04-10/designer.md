## 🖥️ Mockup Wireframe & Visual Blueprint (2장)

### 1. 초기 위험 인지 단계: 데이터 사일로 강조 (The Hook)
*   **목표:** 사용자가 현재의 정보 단절 상태(Pain Point)를 직관적으로 느끼게 하여, EPHA의 개입이 '필요하다'는 위기감을 조성합니다.
*   **레이아웃 콘셉트:** 비효율적인 데이터 흐름을 보여주는 '파편화된 그리드' 형태.

| 요소 | 설명 | 디자인 세부사항 |
| :--- | :--- | :--- |
| **헤더/개요** | 현재 분석 중인 프로세스 및 핵심 지표 요약. | $L_{Total}$ (Placeholder: ???)를 가장 크게 표시하여, '아직 계산되지 않은 손실'이라는 긴장감을 유발합니다. |
| **데이터 사일로 시각화** | 서로 연결되지 않고 분리된 데이터 소스를 개별 그리드 박스 형태로 배치합니다. | 각 박스는 다크 네이비 배경에 불투명한 반투명 경계선만 가집니다. (신뢰감을 주는 미니멀함) <br> **색상:** #1A2333 (딥 네이비) / **텍스트:** #D4E0F5 (라이트 블루) |
| **정보 단절(Gap) 강조** | 사일로와 사일로 사이에 '파단선' 또는 '공백 영역'을 크게 표시합니다. | 이 공백 영역에 빨간색의 희미한 그라데이션(Red Pulse 전 단계)을 입혀, **"무언가 중요한 것이 빠져있다"**는 시각적 압박감을 줍니다. <br> **애니메이션:** 로딩 시작 시, 사일로 간 연결선이 깜빡이며 '신호 없음' 상태를 보여줍니다. |
| **CTA 버튼** | "EPHA 진단 요청: 잠재 손실 측정하기" | 클릭 유도성이 높은 액션 컬러 (시안 블루 강조). #00B8FF (Cyan Blue) 배경, `#FFFFFF` 텍스트. |

### 2. 진단 및 분석 대시보드: Red Pulse 경험 (The Aha Moment)
*   **목표:** $L_{Total}$ 계산 과정(Warning $\to$ Analysis $\to$ Mitigation)을 실시간 스트리밍으로 보여주며, 재무적 손실 발견의 '경험' 자체를 상품화합니다.
*   **레이아웃 콘셉트:** 중앙에 핵심 진단 흐름이 있는 대시보드 형태.

| 요소 | 설명 | 디자인 세부사항 |
| :--- | :--- | :--- |
| **핵심 지표 섹션 (Top)** | $L_{Total}$과 주요 구성요소($L_{Info}, L_{Process}$)를 가장 크고 명확하게 배치합니다. | 수치는 숫자가 아닌, '파도처럼 변화하는 그래프' 형태로 보여줍니다. <br> **애니메이션:** 로딩 직후, $L_{Total}$이 0에서 빠르게 상승하며 불안정한 떨림(Red Pulse의 전조)을 보이다가 안정화됩니다. |
| **진단 여정 플로우 (Center)** | '위험 경고' $\to$ '원인 분석' $\to$ '해결책 제시' 3단계 시퀀스를 시간 축에 따라 배치합니다. | 단계 이동은 단순히 텍스트 변경이 아니라, 빛나는 연결선(Edge)이 다음 블록으로 **강하게 전환되는** 애니메이션을 사용해야 합니다. <br> (기술적 신뢰도를 높이는 핵심 요소) |
| **Red Pulse 경고 모듈** | $L_{Total}$ 임계치 초과 시 발동합니다. 화면 전체를 지배하는 가장 중요한 인터랙션입니다. | 1. 배경의 일부가 강렬한 붉은색(Crimson Red: #CC3333)으로 점진적으로 '파열'되는 듯한 효과 (Dissolve/Pulse). <br> 2. 경고 사유와 구체적인 손실 금액($L_{Total}$)이 큰 글씨로 플래시 됩니다. |
| **개선 기회 제시** | 문제 해결 단계에서, $L_{Info}$를 낮출 수 있는 '핵심 조치' 목록을 카드 형태로 제공합니다. | 이 카드는 시안 블루 계열의 밝고 희망적인 색상으로 배치하여, 경고 모드와 대비되는 '해결책의 빛'을 상징합니다. |

## ⚙️ Interaction Spec Sheet (개발자 전달용)

### A. 핵심 지표 변화 애니메이션 가이드라인

| 상태/변화 | 트리거 | 시각적 효과 (Effect) | 시간 및 속도 (Timing) |
| :--- | :--- | :--- | :--- |
| **L-Cost 산출 시작** | `POST /api/v1/diagnosis/calculate_lcost` 요청 전송. | 중앙 그래프가 0에서 불안정하게 떨리며(Jittering), 배경에 낮은 주파수의 진동 효과(Subtle Pulse)가 추가됨. | 0초 (즉시 시작) $\to$ 5초 동안 지속. |
| **Red Pulse 활성화** | $L_{Total}$ 값이 정의된 임계치($\text{Threshold}_{Crit}$)를 초과할 때. | 배경의 특정 영역(예: 대시보드 상단 30%)이 어두운 네이비에서 강렬한 크림슨 레드(#CC3333)로 급격히 변하며, 화면 전체에 짧고 강력한 진동 효과가 발생함. | **극적 (Dramatic):** 50ms 단위의 깜빡임 + 2초간 유지. |
| **데이터 변화 (정상)** | API 스트리밍 데이터 수신 (`/ws/risk/stream`). | 숫자가 부드럽게(Easing Out) 증가하거나 감소하는 애니메이션을 적용합니다. 급격한 점프는 피합니다. | 100ms ~ 300ms 간의 Smooth Transition. |
| **지식 연결 (Aha Moment)** | 진단 여정 플로우가 '분석' $\to$ '해결책'으로 전환될 때. | 이전 단계의 노드-엣지가 빛을 내며(Glow Effect), 다음 해결책 영역으로 강력하고 명확하게 빨려 들어가는 듯한 **트랜지션 모션**을 구현합니다. | 1초에 걸친 역동적인 시각적 흐름 (Flow). |

### B. 핵심 인터랙티브 요소 (UX Interaction Details)

| 기능/요소 | 사용자 액션 | 시스템 응답 및 애니메이션 | 비고 (개발 지시사항) |
| :--- | :--- | :--- | :--- |
| **진단 시작** | CTA 버튼 클릭. | 1. 초기 로딩 시, "데이터 파편 분석 중..." 문구와 함께 사일로 그리드가 빠르게 스캔되는 애니메이션을 보여줍니다. (지연시간 부여) <br> 2. $L_{Total}$ 그래프가 불안정하게 진동하며 Red Pulse 전조를 보입니다. | **필수:** 이 단계에서 백엔드 `POST` 요청을 보내고, 동시에 WebSocket 연결(`WS`)을 시작해야 합니다. |
| **Red Pulse 경고** | (자동 트리거) $L_{Total}$이 임계치를 초과할 때. | 1. Red Pulse 발동 $\to$ 화면 중앙에 '위험!' 배너가 오버레이됩니다. <br> 2. 사용자의 포커스가 강제적으로 가장 중요한 지표($L_{Total}$)로 이동하며, 주변 요소는 어둡게 처리(Dimming) 됩니다. | **강조:** 이 모드에서는 모든 인터랙션이 '위험 해결'에 초점을 맞춰야 합니다. |
| **해결책 제시** | 분석 과정에서 핵심 조치를 클릭할 때. | 해당 조치($\text{Mitigation}$)가 가상의 데이터 파편(Input Data)으로 재주입되는 시뮬레이션을 짧게 보여줍니다. (애니메이션: 사일로 $\to$ 연결선). <br> 이후, $L_{Total}$이 감소하는 애니메이션과 함께 '예상 절감액'을 수치적으로 보여줍니다. | **성과:** 이 과정은 사용자에게 "우리가 이걸 할 수 있다"는 확신(Confidence)을 주어야 합니다. |
