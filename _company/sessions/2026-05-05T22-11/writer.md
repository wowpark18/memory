## 🛠️ EPHA 핵심 사용자 스토리 (User Stories)
*개발팀 백로그용 포맷*

### 📄 User Story 1: 규제 변화 리스크 사전 감지 및 최소화 (Compliance Risk Mitigation)
이 스토리는 **법적/운영적 위험 감소율**이라는 최고 가치에 집중합니다. 정보 단절의 결과가 '벌금'이나 '소송'과 같은 명확한 금전적 손실일 때를 가정합니다.

| 구분 | 상세 내용 |
| :--- | :--- |
| **액터 (As a)** | 기업 컴플라이언스 담당자 / 법무팀 (Legal Officer) |
| **요구사항 (I want to)** | 현재의 내부 프로세스(SOP)가 최신 규제 변화 문서와 충돌하거나 누락된 지점을 **자동으로 연결하여 시각화**하고, 이로 인해 발생할 수 있는 잠재적 법적 리스크를 **정량적인 점수(Risk Score)**로 확인하고 싶다. |
| **핵심 기능 (Feature Focus)** | 1. `Regulatory Corpus` 기반의 실시간 규제 트래킹 모듈. <br> 2. 기존 SOP와 새 규제 간의 `Conflict` 연결을 감지하는 로직 및 경고 알림(`riskScore > Threshold`). |
| **측정 가능 결과값 (Value)** | "EPHA를 사용함으로써, 기존 매뉴얼만으로는 발견할 수 없었던 **잠재적 벌금 규모 $X$ 원**의 리스크(Risk Gap)를 사전에 제거하고 준수율을 $\text{Y}\%$까지 높일 수 있다." |

### 📄 User Story 2: 의사결정 병목 구간 해소 및 속도 가속화 (ATD Reduction & Bottleneck Removal)
이 스토리는 **의사결정 시간 단축률**에 집중합니다. 정보가 부족해서(Gap) 프로젝트가 지연되는 '시간 비용'을 줄이는 데 초점을 맞춥니다.

| 구분 | 상세 내용 |
| :--- | :--- |
| **액터 (As a)** | 대형 프로젝트 매니저 / 팀 리더 (Project Manager) |
| **요구사항 (I want to)** | 복잡한 다부서 프로젝트의 의사결정 과정(Workflow Log)에서, 정보 부족으로 인해 **불필요하게 지연되는 병목 구간(Bottleneck Node/Edge)**을 식별하고, 이 Gap을 해소하는 데 필요한 필수 정보를 즉시 찾아 연결하여 프로세스 흐름도를 재구성하고 싶다. |
| **핵심 기능 (Feature Focus)** | 1. 시간 순서 기반의 워크플로우 시각화 및 지연 구간(Delay Spot) 하이라이트. <br> 2. '누락된 필수 정보'를 자동으로 예측하여 연결 경로를 제시하는 추천 엔진(`connectionType=SUPPLEMENTARY` + `gapDetected`). |
| **측정 가능 결과값 (Value)** | "EPHA 덕분에, A 단계와 B 단계 사이에 발생하던 평균 $\text{Z}$ 일의 대기 시간이 절약되어 프로젝트 완료 시점을 최소 $\text{W}\%$ 앞당길 수 있다." |

***

### 📄 User Story 3: 지식 연결 과정 탐색을 통한 통찰 발견 (Contextual Insight Discovery)
이 스토리는 정보 그 자체가 아니라 **정보를 얻는 '과정'** 자체의 가치를 사용자에게 제공합니다. 연구자나 기획자가 필요로 하는 'Aha Moment'에 초점을 맞춥니다.

| 구분 | 상세 내용 |
| :--- | :--- |
| **액터 (As a)** | 시장 분석가 / 전략 컨설턴트 (Strategist) |
| **요구사항 (I want to)** | 특정 키워드 검색(A)으로 인해 정보에 막히거나 맥락을 잃는 상황(Pain Point)에서, EPHA 시스템이 해당 지식과 관련성이 높지만 **직접 연결되지 않은 주변 지식(Peripheral Knowledge)**들을 '정보 흐름 지도' 형태로 제시하여, 사용자가 예상치 못한 새로운 관점이나 통찰(Insight)에 도달하고 싶다. |
| **핵심 기능 (Feature Focus)** | 1. 검색어 중심이 아닌, 개념(Concept) 중심으로 노드 그래프를 확장하는 인터랙티브 탐색 모드. <br> 2. 사용자가 현재까지의 경로와 연결되지 않은 '연결 가능성'을 점수화하여 제시(`riskScore`가 낮고 `knowledgeLevel`이 높은 주변 Node). |
| **측정 가능 결과값 (Value)** | "기존 검색 방식으로는 얻지 못했던, A와 C라는 전혀 다른 분야의 지식을 연결하는 새로운 가설적 관계를 발견하여, 시장 분석 보고서에 활용할 수 있는 **독창적인 인사이트 3가지**를 도출했다." |
