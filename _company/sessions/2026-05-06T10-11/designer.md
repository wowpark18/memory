# ✨ 인터랙티브 모킹 컴포넌트 설계 브리프 (3종)

## [전반적인 디자인 원칙]
*   **톤앤매너:** DeepTech, 구조적, 명료함.
*   **핵심 경험:** 사용자가 수동적으로 결과를 보는 것이 아니라, EPHA의 '지능'이 작동하는 것을 **관찰(Observe)**하게 만들어야 합니다.
*   **색상 팔레트 매핑 (State-based):**
    *   **Pain/Gap Discovery:** `#D9534F` (Warning Red) - 경고, 위험, 단절.
    *   **Process/Interaction:** `#007bff` (Primary Blue) - 분석, 흐름, 지능적 연결.
    *   **Mitigation/Success:** `#5cb85c` (Success Green) - 해소, 완결성, 절감 가치.

### 1. Component: The Gap Visualizer (정보 단절 리스크 경고)
**(Pain Point / $F_{Gap}$ 발견 순간)**

| 요소 | 세부 내용 및 기능 명세 | 인터랙션/애니메이션 콘셉트 |
| :--- | :--- | :--- |
| **컴포넌트명** | **Dynamic Connection Graph (불안정 노드 맵)** | 사용자가 초기 데이터셋을 업로드했을 때, 즉시 작동합니다. |
| **목적** | 데이터 간의 '정보 단절' 지점을 시각적으로 파괴된 형태로 제시하여 위기감을 극대화하고, $F_{Gap}$가 발생하는 근본 원인을 직관적으로 인지시킨다. |
| **구조/레이아웃** | 1. 중앙에 주요 노드(Node)들이 무질서하게 배치됨. 2. 대부분의 연결선(Edge)은 정상적인 파란색(`Primary Blue`)으로 표시되나, 핵심 프로세스 단절 지점만 `Warning Red`로 깜빡임. 3. 단절된 Edge 위에는 'Gap Reason: [원인 명시]'라는 경고 모달이 오버레이됨. |
| **핵심 애니메이션** | - **"끊어짐 효과 (Disconnection Pulse)":** $F_{Gap}$가 계산되는 순간, 특정 노드 쌍 사이의 연결선(Edge)들이 붉은색 파동과 함께 떨리며 끊어지는 모션(`Red Fade Out`)을 보여줍니다. <br>- **강조:** 단절된 지점을 중심으로 주변 노드들이 마치 공명하는 것처럼 미세하게 진동하며 불안정성을 표현합니다. |
| **출력 값** | 계산된 $F_{Gap}$ (예: "Potential Loss: $1,234,560원")을 빨간색 박스에 크고 경고적으로 표시. |

### 2. Component: The Edge Tracer (정보 단절 해소 프로세스)
**(Process / 단계별 분석 과정)**

| 요소 | 세부 내용 및 기능 명세 | 인터랙션/애니메이션 콘셉트 |
| :--- | :--- | :--- |
| **컴포넌트명** | **Linear Timeline Stepper (프로세스 플로우 트래커)** | 사용자가 '분석 실행' 버튼을 클릭할 때 작동하는 핵심 컴포넌트. |
| **목적** | $F_{Gap}$가 어떻게 계산되고, EPHA의 프로세스가 어떤 단계를 거쳐 이를 메우는지(Edge)를 시간 순서대로 보여주어 신뢰도를 높인다. |
| **구조/레이아웃** | 1. 화면 중앙에 수평형 타임라인을 배치합니다. (최소 3~5단계로 분할). 2. 각 단계는 제목, 설명(텍스트), 그리고 해당 단계에서 사용되는 데이터 유형(예: '법규 문서' $\rightarrow$ '내부 보고서')으로 구성됩니다. 3. 각 단계의 상단에는 현재 진행 중인 프로세스 흐름을 나타내는 지표(Progress Bar)가 표시되어야 합니다. |
| **핵심 애니메이션** | - **"빛나는 경로 (The Edge Flow)":** 사용자가 다음 단계를 클릭하거나 시스템이 자동으로 이동할 때, 이전 단계에서 다음 단계로 넘어가는 연결선(`Primary Blue`) 자체가 마치 빛을 내며(Glowing Effect) 트레일링 되는 모션을 보여줍니다. <br>- **시각화:** 각 단계를 거치면서 초기 노드(Node)가 점점 더 구조화되고 명확해지는 애니메이션이 필수적입니다. (혼돈 $\rightarrow$ 질서). |
| **사용자 액션** | 단계별 설명을 읽는 동안, 옆의 작은 'Detail View'에서 해당 단계에 사용된 원본 데이터 조각들을 팝업으로 보여주어 투명성을 확보합니다. |

### 3. Component: The Value Transformer (절감 가치 및 해결 완료)
**(Mitigation / $\text{Mitigated Value}$ 확인 순간)**

| 요소 | 세부 내용 및 기능 명세 | 인터랙션/애니메이션 콘셉트 |
| :--- | :--- | :--- |
| **컴포넌트명** | **Dual-Axis Impact Dashboard (가치 대비 그래프)** | 최종 결과 화면에 배치되어, 문제와 해결의 크기를 동시에 비교한다. |
| **목적** | 사용자가 느꼈던 손실($F_{Gap}$)이 얼마나 큰 액수의 절감 가치($\text{Mitigated Value}$)로 전환되었는지 직관적으로 증명하고 성공 경험을 제공한다. |
| **구조/레이아웃** | 1. 화면 중앙에 두 개의 막대 그래프(Bar Chart)를 나란히 배치합니다. (Dual Axis). <br>2. 왼쪽 바: $F_{Gap}$ (빨간색, 최대 손실액). <br>3. 오른쪽 바: $\text{Mitigated Value}$ (녹색, 절감 가치). 4. 두 막대 그래프 위에 **'Loss Reduction Ratio ($\%)$'**라는 핵심 지표를 큰 숫자로 표시합니다. |
| **핵심 애니메이션** | - **"가치 전이 효과 (Transformation Sweep)":** 컴포넌트 로딩 시, 빨간색의 $F_{Gap}$ 막대가 화면 아래에서 위로 상승하다가(위험), 녹색 $\text{Mitigated Value}$ 막대와 만나면서 그 에너지가 긍정적인 힘으로 전환되는 듯한 부드러운 트랜지션 애니메이션을 구현합니다. <br>- **피드백:** 최종 값이 확정되면, 그래프 주변에 미세하고 밝은 녹색의 'Success Particle' 효과가 퍼져나갑니다. |
| **핵심 메시지** | "잠재적 손실액 $F_{Gap}$ (빨간색) $\rightarrow$ EPHA 프로세스 적용 $\rightarrow$ 절감 가치 $\text{Mitigated Value}$ (녹색)"라는 흐름을 한눈에 담는다. |
