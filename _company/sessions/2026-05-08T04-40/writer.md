### ⚙️ PoC 더미 데이터 시나리오: 위험/손실 발견 (2가지)

#### 🔴 [시나리오 1] 제조업 - 예측 정비(PdM) 및 공급망 리스크
*   **산업:** 중공업 / 스마트 팩토리
*   **핵심 손실 지표:** $L_{Info}$ (정보 단절로 인한 운영 비효율성 및 가동 중단 비용)

##### **1. 위험 발생 스토리 (Risk Occurrence Story)**
| 필드 | 내용 | 개발자 바인딩 참고 사항 |
| :--- | :--- | :--- |
| **제목 (Headline)** | [경고: Red Pulse] 핵심 생산 라인 'A-203'의 갑작스러운 가동 중단 - 원인은 단순 고장이 아니다. | *[강렬한 톤앤매너, 긴급 알림 스타일]* |
| **요약 (Summary)** | 평소 안정적이었던 '특수 CNC 공작기계 A-203'이 예상치 못한 주간에 가동을 멈췄습니다. 정비팀은 단순 베어링 마모로 판단했지만, 실제 데이터는 이전에 기록된 *재료 변화*와 *최대 부하 패턴* 간의 단절을 보여주고 있었습니다. | *[Pain Point 강조]* |
| **문제 정의 (Problem Description)** | 공작기계 A-203은 지난 6개월간 주력 재질이 '알루미늄 합금'에서 '티타늄 코팅 강철'로 변경되었습니다. 하지만 이 재료 변화에 따른 *진동 패턴*과 *최적 속도 범위*의 조정 기록이 정비 매뉴얼 및 IoT 센서 데이터 사이에 **정보 사일로(Information Silo)** 형태로 존재했습니다. | *[데이터 단절/사일로 강조]* |
| **예상 손실액 (Estimated Loss)** | 주당 예상 생산량 기준, 최소 300만원 / 일 평균 가동 중단 시 $\text{Min } \$15,000 \sim \$25,000$ | *[재무적 언어 필수]* |

##### **2. 손실 발견 및 해결 과정 (Loss Discovery & Solution Flow)**
| 필드 | 내용 | 개발자 바인딩 참고 사항 |
| :--- | :--- | :--- |
| **발견 지표 (Key Metric)** | $\mathbf{L_{Info}}$ (정보 연결성 결여에 따른 손실): 공작기계 재질 변화 데이터와 실시간 진동 데이터를 연결하지 못한 비용. | *[핵심 로직 지표]* |
| **진단 과정 (Diagnostic Flow)** | EPHA 시스템이 1) '재료 변경 시점'을 파악 → 2) 해당 기간의 '이상 진동 패턴 임계치'를 계산 → 3) 현재 센서 데이터가 이 임계치를 벗어남을 시각화하여 경고. (정비 매뉴얼과의 교차 검증 필요). | *[Step-by-step Flow]* |
| **해결책 및 통찰 (Actionable Insight)** | 기계적 고장이 아닌, 새로운 재료가 발생시키는 '미세 진동 주파수 변화'에 대한 사전 예측 정비(Predictive Maintenance)가 필요합니다. 공정 데이터를 통합하고, 재질별 최적 가공 매뉴얼을 시스템화해야 합니다. | *[Solution/Next Step 제시]* |
| **사용자 액션 (User Action)** | 🚨 [위험 경고] → **데이터 연결성 시뮬레이터** 실행 → $\text{Optimal\_Speed}$ 값 자동 재조정 및 공정 개선안 도출. | *[UI 버튼/상호작용 요소]* |

#### ⚡️ [시나리오 2] 유틸리티/에너지 - 그리드 부하 및 컴플라이언스 리스크
*   **산업:** 에너지 / 전력 공급 (Smart Grid)
*   **핵심 손실 지표:** $L_{Total}$ (예측 실패로 인한 직접적 재무 손실 + 벌금/위약금)

##### **1. 위험 발생 스토리 (Risk Occurrence Story)**
| 필드 | 내용 | 개발자 바인딩 참고 사항 |
| :--- | :--- | :--- |
| **제목 (Headline)** | [긴급!] 예측치 못한 전력 부하 과부하 임계점 도달 - 재해 예방 실패 비용. | *[위협적, 공공의 안전과 직결]* |
| **요약 (Summary)** | 평소에는 안정적이던 지역 변전소가 갑작스러운 '이상 기상' 조건(예: 폭우와 함께 예상치 못한 냉난방 수요 증가)으로 인해 과부하 위험에 처했습니다. 기존 시스템은 이 복합적 요소를 예측하지 못했습니다. | *[Pain Point - 복합성 강조]* |
| **문제 정의 (Problem Description)** | 전력 부하는 단순히 '현재 사용량'만 보는 것이 아니라, 1) 실시간 기상 데이터(온도 변화 속도), 2) 역사적 재난 패턴(과거 폭우 시의 수요 급증 정도), 3) 지역별 인구 밀집도와 결합된 **다차원 예측 모델**이 필요합니다. 이 변수들 간의 상호작용 지표가 누락되었습니다. | *[복잡한 데이터 통합 요구]* |
| **예상 손실액 (Estimated Loss)** | 1) 예상 다운타임 벌금 ($\text{Penalty}$): $\text{Min } \$5M$. 2) 기회비용($\text{Opportunity Cost}$): 추정치 $\text{Min } \$8M \sim \$12M$ | *[재무적 손실의 복합성 강조]* |

##### **2. 손실 발견 및 해결 과정 (Loss Discovery & Solution Flow)**
| 필드 | 내용 | 개발자 바인딩 참고 사항 |
| :--- | :--- | :--- |
| **발견 지표 (Key Metric)** | $\mathbf{L_{Total}}$ (총체적 손실): 기상 예측 데이터와 부하 패턴 데이터를 통합하여 '임계점 도달 확률'을 측정. | *[가장 포괄적인 리스크 관리 지표]* |
| **진단 과정 (Diagnostic Flow)** | EPHA 시스템이 1) 실시간 $\text{Weather API}$ 데이터 입력 → 2) 과거 $\text{Historical Load Data}$와 비교하여 '최대 위험 계수(Hazard Coefficient)' 산출 → 3) 임계점 도달까지 남은 시간과 예상 최대 부하를 시각화. | *[API 연동 및 예측 로직 강조]* |
| **해결책 및 통찰 (Actionable Insight)** | 단순히 전력을 증설하는 것만으로는 부족합니다. 기상 변화와 부하 패턴을 실시간으로 통합 분석하여, '어느 구역'의 전력 흐름을 가장 먼저 분산시켜야 하는지(Load Balancing Priority)에 대한 **선제적 결정권**이 필요합니다. | *[결정 권한 부여 강조]* |
| **사용자 액션 (User Action)** | 🚨 [위험 경고] → **부하 시뮬레이터** 실행 → 가장 위험도가 높은 세 개의 분산 경로를 선택하고, 실시간 자원 재배치를 명령한다. | *[최종 의사결정 및 제어권 강조]* |
