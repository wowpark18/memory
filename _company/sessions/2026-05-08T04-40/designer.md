## 1. 위험 지표 표시기 컴포넌트: Risk Gauge Component

**[목적]** 특정 데이터 사일로 또는 프로세스 단계에서 발생하는 '위험도'를 직관적인 게이지 형태로 시각화합니다. (정량적 측정값을 감성적으로 변환)
**[크기/레이아웃]** 가로 300px x 세로 150px (반원형 그래프 형태 권장).

| 속성 | 스펙 및 가이드라인 | 상호작용(Interaction) | 개발 참고 사항 |
| :--- | :--- | :--- | :--- |
| **구조** | 1. 게이지 원형 영역 (Radial Bar) / 2. 수치 표시기 (Numerical Text) / 3. 위험 레벨 레이블 (Text Label) | **Hover State:** 마우스 오버 시, 게이지가 천천히 채워지며(Smooth Transition), '데이터 근거' 요약 정보 패널이 하단에 나타남. | *Pseudo-Code:* `gauge_fill_percent` (0~100). 이 값이 주요 상태를 결정함. |
| **색상 로직** | 🟢 (Green: 0-25%) $\rightarrow$ 🟡 (Yellow: 26-75%) $\rightarrow$ 🔴 (Red: 76-100%) 순으로 그라데이션 적용. | **Trend Indicator:** 게이지 옆에 작은 화살표(▲/▼)를 배치하여 전 주기 대비 변화율을 보여줌. (예: ▼ -2% / ▲ +5%). | `risk_level` 값에 따라 `Crisis-Red`와 `Highlight-Blue`의 경계선 처리가 필요함. |
| **데이터 바인딩** | 예시: `현재 위험도`: 85% $\rightarrow$ (🔴) 높은 수준. <br> *설명:* 프로세스 단계 간 데이터 연결성 부족으로 인한 잠재적 손실 우려. |

## 2. $L_{Total}$ 수치 카드 컴포넌트: Loss Metric Card

**[목적]** EPHA가 발견한 '측정 가능한 재무적 손실'의 규모를 C-Level이 즉시 이해할 수 있도록 강력하게 제시합니다. (숫자의 충격적인 시각화)
**[크기/레이아웃]** 가로 400px x 세로 200px. 데이터 중심의 카드형 구조.

| 속성 | 스펙 및 가이드라인 | 상호작용(Interaction) | 개발 참고 사항 |
| :--- | :--- | :--- | :--- |
| **구조** | 1. 핵심 지표 제목 (Metric Title) / 2. **손실 금액 (Loss Amount)** (가장 크게, 가장 강조되는 요소) / 3. 손실 발생 원인 요약 (Source/Context). | **Tooltip:** 카드 위에 커서가 올라가면, 이 수치가 산출된 *세부 데이터 로그*를 담은 작은 모달 창이 나타나야 함. | `Loss Amount`는 통화 기호($)와 함께 가장 큰 폰트 크기(H1/2)로 처리해야 합니다. |
| **색상 로직** | 손실 금액(`L_{Total}`) 자체는 배경과 대비되게 어두운 `#333` 계열의 강조 색상으로, 전반적인 '위험'을 나타내야 함. 비교 지표(예: 기대 비용)는 `Highlight-Blue`를 사용해 대조 효과를 극대화. | **Toggle View:** 카드 하단에 [Period/Metric] 버튼이 있어, 기간별 손실액 또는 다른 핵심 KPI($L_{Info}$)로 빠르게 전환 가능해야 함. | `$Loss Amount`: 굵은 글씨체(Bold) 필수. 데이터의 크기를 과장하여 보여주는 것이 목표입니다. |
| **데이터 바인딩** | 예시: $\mathbf{\$25,000}$ (주당 예상 손실액). <br> *발견 지표:* $L_{Info}$ (정보 단절) / *비교*: 기대 가동 중단 비용($\$15,000$) 대비 67% 증가. |

## 3. Red Pulse 경고 애니메이션 컨테이너: Crisis Alert Container

**[목적]** 시스템이 치명적인 '이상 패턴'을 포착했을 때의 강렬한 시각적/경험적 경험(Aha Moment)을 제공합니다. 단순 알림창이 아닌, *사건 발생* 그 자체를 보여줍니다.
**[크기/레이아웃]** 화면 전체 영역에 오버레이되거나 (Overlay), 특정 데이터 포인트 주변에서 폭발적으로 발생해야 합니다.

| 속성 | 스펙 및 가이드라인 | 상호작용(Interaction) | 개발 참고 사항 |
| :--- | :--- | :--- | :--- |
| **구조** | 1. 경고 메시지 (Headline - 충격적 질문 형식) / 2. Pulse 영역 (애니메이션의 중심) / 3. 근거 데이터(Data Trail). | **Animation Flow:** `[Trigger] $\rightarrow$ [Pulse Expansion] $\rightarrow$ [Fade Out & Diagnosis]` 순서로 진행되어야 합니다. 애니메이션이 가장 중요함. | 이 컴포넌트는 CSS/WebGL 기반의 고성능 그래픽 처리가 요구됩니다. |
| **애니메이션** | 1. **Red Pulse:** 중앙 포인트에서 시작하여 빨간색 에너지 파동(Radial Wave)이 전방위로 퍼져나가는 효과. (지진파/음파 느낌). <br> 2. **Glitch Effect:** 경고 메시지가 나타날 때, 짧은 순간 디지털 노이즈나 화면 왜곡(Glitch) 효과를 삽입하여 위기감을 고조시킵니다. | **Persistent State:** 애니메이션 종료 후에도 주변 컴포넌트들이 미세하게 떨리는(Jittering) 잔상을 남겨 '여전히 위험함'을 암시해야 합니다. | `Red Pulse`의 파동은 배경색과 대비되도록 투명도(`opacity`)를 단계적으로 변화시켜야 함. |
| **활성화 트리거** | $L_{Info}$ 임계치 초과 $\rightarrow$ 시스템이 기존 데이터 세트와 새로운 데이터를 연결하려 실패했을 때 (Failure Point). |
