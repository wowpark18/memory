## 🚨 EPHA 인터랙티브 데모 UI/UX 상세 명세서: [L-Total Crisis Simulation]

본 문서는 Developer가 정의한 WebSocket 기반의 실시간 데이터 스트리밍 시뮬레이터 엔진을 완벽하게 시각화하고, 사용자가 '정보 단절로 인한 재무적 위기감'을 극대하게 체험할 수 있도록 하는 고정밀 디자인 블루프린트입니다.

---

### 1. 전반적인 디자인 원칙 및 목표 감성 (Tone & Emotion)

*   **핵심 경험:** '안정 $\rightarrow$ 비효율 감지 $\rightarrow$ 정보 단절 발생 $\rightarrow$ 위기 체험'의 서사적 흐름을 시각화한다.
*   **톤앤매너:** 딥테크(Deep-Tech), 클리니컬(Clinical), 긴장감 고조형 (High Tension).
*   **색상 활용 원칙:** 색상은 단순히 '경고'를 넘어, 시스템의 **정상 작동 여부**와 **정보 접근성의 상태**를 나타내는 지표여야 한다.
    *   🟢 **Safe State (안전):** 신뢰감을 주는 딥 네이비/블랙 배경에 안정적인 시안 블루($\text{#4A90E2}$) 사용.
    *   🟡 **Warning State (주의):** 데이터 변동성이 높아지며, 노란색($\text{#F5A623}$)과 주황색 경계가 나타난다.
    *   🔴 **Crisis State (위기/정보 단절):** $\text{Red Pulse}$ ($\text{#D92A2C}$). 이 색상은 데이터 오류, 연결 실패, 정보 부족을 상징하며, 화면의 모든 요소를 침범해야 한다.

### 2. 핵심 컴포넌트 상세 명세 (Component Specification)

#### A. $L_{Total}$ 계기판 (Gauge Display)
| 속성 | Safe State ($\text{L}_{Total} < 60$) | Warning State ($60 \le \text{L}_{Total} < 90$) | Crisis State ($\text{L}_{Total} \ge 90$ 또는 임계점 초과) |
| :--- | :--- | :--- | :--- |
| **시각적 표현** | 부드러운 아날로그 바늘 움직임. 배경에 미세한 녹색 그리드가 흐름. | 바늘이 흔들리는 듯한(jittering) 애니메이션 적용. 주변에 주황색 노이즈 오버레이 추가. | **Red Pulse 효과:** 계기판 전체가 붉은 깜빡임($\text{#D92A2C}$)을 반복하고, 값이 불안정하게 급등락한다. 바늘 움직임 자체가 떨리며(vibration), 아날로그적 신뢰도가 무너지는 느낌을 준다. |
| **데이터 표시** | 큼직한 디지털 숫자와 함께, 추세선 (Trend Line)이 부드럽게 그려진다. | 숫자가 미세하게 '떨리는' 효과를 주어 불안정함을 강조한다. | 숫자가 일시적으로 깨지거나(glitch), 비정상적인 오류 코드가 빠르게 지나가는 애니메이션을 삽입한다. |

#### B. 데이터 흐름 그래프 (Knowledge Flow Graph)
이 컴포넌트는 시스템의 '건강 상태'를 시각화하는 가장 중요한 영역이다.

1.  **노드(Nodes):** 각 지식/데이터 개념을 나타낸다.
    *   *Safe State:* 노드는 밝은 파란색($\text{#4A90E2}$)으로 빛나며, 내부에는 미세한 '활성화' 애니메이션(pulsing)이 적용된다.
    *   *Crisis State:* 연결성이 끊어진 노드 주변은 어둡게 처리되고, 해당 노드의 이름 위에 `[DATA GAP]` 또는 `[UNKNOWN VALUE]`라는 붉은 라벨이 오버레이된다.

2.  **엣지(Edges):** 지식 간의 관계성/정보 흐름을 나타낸다.
    *   *Safe State:* 부드럽고 일정한 빛나는 시안 블루 그리드 연결선. 마치 안정적인 데이터 파이프라인처럼 보인다. (속도: 1초에 걸쳐 완만한 이동)
    *   *Warning State:* 간헐적으로 노란색의 지연(latency) 효과가 나타나며, 선의 색상이 주황빛을 띠기 시작한다.
    *   **Crisis State:** **정보 단절 시뮬레이션 (The Red Pulse):** 연결선이 불규칙하게 끊어지고($\text{Broken Edge}$), 파열되는(shattering) 애니메이션과 함께 빨간색 노이즈가 전파된다. 이는 데이터 흐름의 구조적 실패를 의미하며, 사용자에게 **'핵심 정보에 접근할 수 없음'**이라는 심리적 압박을 가한다.

#### C. 위기 알림 컴포넌트 (System Alert Modal)
*   **발동 조건:** $\text{L}_{Total}$이 특정 임계점(예: 90 이상)에 도달하고, 동시에 정보 단절(Red Pulse)이 감지될 때만 발동한다.
*   **디자인:** 화면의 중앙 상단에 위치하며, 배경은 어둡게 흐려지고 (Dimming Effect), 이 모달창만이 $\text{Red Pulse}$ 색상으로 강조되어 나타난다.
*   **카피라이팅 (예시):** "⚠️ **WARNING: CRITICAL INFORMATION GAP DETECTED.** 시스템 마찰 비용($\text{L}_{Total}$)이 위험 임계점을 초과했습니다. 현재 데이터 플로우의 $\text{X}$ 지점에서 구조적 정보 단절(Information Void)이 감지됩니다. 근본 원인 분석을 위해 $Y$ 액션이 필수입니다."

### 3. 애니메이션 및 인터랙션 상세 가이드 (Motion & Interaction Guide)

| 상호작용/상태 변화 | 목표 효과 | 기술 명세 (Timing/Easing) |
| :--- | :--- | :--- |
| **데이터 수신 (Streaming)** | 실시간, 연속적인 데이터 흐름의 신뢰성 강조. | 100ms 단위로 그래프 요소가 아주 미세하게 위치 조정됨 (Jittering). CSS `ease-in-out`을 사용하여 부드러운 움직임을 유지한다. |
| **위기 감지 (Threshold Cross)** | 평온함 $\rightarrow$ 혼란으로의 급격한 심리적 전환 유도. | 0.5초 이내로 색상 팔레트 전체가 시안 블루에서 주황/빨강 계열로 급변(Flash Transition). 사운드적으로는 저주파 진동음(Deep Sub-Bass)을 동반해야 한다. |
| **정보 단절 (Red Pulse)** | 구조적 실패와 공포의 극대화. | 1~2초 주기의 빠르고 불규칙한 깜빡임(Flicker/Strobe Effect). 파열 애니메이션은 노드에서 시작하여 주변 그리드로 확산되는 '파동 효과'를 사용한다. (CSS `keyframes` 기반) |
| **사용자 조작 (Input)** | 사용자에게 '개입의 필요성'을 부여. | 사용자가 어떤 지점에 마우스를 올리거나 버튼을 클릭하면, 해당 노드/엣지는 일시적으로 밝고 안정적인 파란색으로 빛나며(Focus Glow), 잠재적 해결 경로를 제시하는 시각적 확신을 준다. |
