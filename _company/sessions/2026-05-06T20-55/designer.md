# 🚨 Gap Alert 컴포넌트 마이크로 인터랙션 가이드라인 (PoC 버전)

본 가이드는 Developer의 API 로직(Detection $\rightarrow$ Scoring $\rightarrow$ Solution)과 Business가 정의한 사용자 경험 플로우를 결합하여, 사용자의 몰입도와 긴급성을 극대화하는 세부 비주얼 및 인터랙션 디렉션을 제공합니다. 모든 상호작용은 '정보 단절'의 물리적/개념적 체험에 중점을 둡니다.

## 🖼️ 공통 디자인 원칙 (Design System 적용)

| 요소 | 상태 | 색상 코드 (HEX) | 애니메이션 컨셉 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **기본 배경** | 정상 흐름 | `#0A192F` (딥 네이비/블랙 기반) | 부드러운 그리드 오버레이 애니메이션 (미세한 빛의 떨림) | 신뢰감, 기술적 깊이 유지. |
| **Gap 발생 지점** | 경고(Warning) | `#FF4D4D` (강렬한 레드) + 투명도 조절 | Glitch Effect (데이터 왜곡/파열), 빨간색 깜빡임 (Flashing) | 흐름의 강제적 중단 시각화. |
| **연결 과정** | 발견(Discovery) | `#4D99FF` (시안 블루) + 그라데이션 | 빛나는 엣지(Glowing Edge)가 노드를 따라 움직이는 트랜지션 | 희망, 연결성, 통찰의 순간. |
| **해결책 제시** | 안내(Guidance) | `#6AA84F` (차분한 그린/골드톤) | 부드러운 페이드인(Fade-in), 명확하게 포커싱되는 효과 | 안정감, 실행 가능성 부여. |

## 🕹️ 단계별 마이크로 인터랙션 디렉션

### 1단계: Gap 발생 감지 (The Problem - `POST /detect_gap` 트리거)
**목표:** 사용자가 '지금 뭔가 잘못되었다'는 직관적이고 강렬한 위기감을 느끼게 함.

*   **트리거:** 개발자가 데이터를 입력(Input Data Point)하고, 백엔드에서 $F_{Gap}$가 탐지되는 순간.
*   **애니메이션 (Glitch Warning):**
    1.  사용자가 작업하던 화면 전체의 배경 그리드가 갑자기 **붉은색 노이즈 필터(Red Noise Filter)**로 덮입니다.
    2.  데이터 흐름을 나타내는 모든 요소(엣지)가 순간적으로 **좌우/상하로 '찢어지는(Tearing)'** 듯한 글리치 효과를 보여줍니다. (Duration: 0.5초).
    3.  화면 중앙에 "🚨 프로세스 무결성 손실 감지 ($F_{Gap}$ Detected)"라는 경고 배너가 **강렬하게 깜빡이며** (Flashing Red, 1.2Hz) 나타납니다.
*   **UX 피드백:** 사용자 커서의 이동이 일시적으로 제한되며, 화면 가장자리에서 낮은 주파수의 **'경고음(Low-frequency warning tone)'** 사운드가 재생됩니다.
*   **컴포넌트 변화:** Gap 발생 지점과 관련된 노드는 붉은색 테두리(`Red Border`)가 생기며, 다른 정상 노드들과의 연결선(Edge)이 흐릿하게 **'끊어진'** 것처럼 시각화되어야 합니다.

### 2단계: 위험도 분석 및 정량화 (Analysis - `POST /score_risk` 트리거)
**목표:** 막연한 위협을 구체적인 '재무적/운영적 손실 비용(L-Cost)'으로 전환하여 문제의 심각성을 체감하게 함.

*   **트리거:** Gap이 감지된 후, Risk Scoring Service가 L-Cost를 계산하는 과정 시작.
*   **애니메이션 (Data Surge):**
    1.  화면이 잠시 어두워지고(Dimming), 배경에 **'데이터 분석 중...'**이라는 문구가 나타납니다.
    2.  L-Cost 측정 지표가 그래프 형태로 등장하며, 수치가 초당 10~20 단위로 빠르게 계산되어 올라갑니다 (Data Surge Effect). 이 과정에서 시안 블루 계열의 빛나는 선이 데이터 포인트를 따라 움직이며 '분석' 과정을 시각화합니다.
    3.  최종 L-Cost 수치(`$X Million Loss`)가 등장할 때, 해당 숫자가 **잠시 동안 화면 전체를 덮으며(Full Screen Overlay)** 가장 크게 강조됩니다. (이때의 강렬한 크기 변화와 진동 효과가 중요).
*   **UX 피드백:** 사용자에게 "Gap으로 인한 잠재적 손실 규모: $X Million"이라는 메시지와 함께, **'문제 해결을 위한 행동이 시급합니다.'**라는 톤앤매너를 유지하는 안내 문구를 제공합니다.

### 3단계: 해결책 제시 및 프로세스 수정 (Solution Integration - User Action)
**목표:** 혼란에서 명확한 해답으로의 전환(Aha Moment)을 극적으로 연출하며 사용자의 행동을 유도함.

*   **트리거:** 시스템이 최적의 워크플로우를 제안하고, 사용자에게 '수정/채택' 버튼을 제시할 때.
*   **애니메이션 (Guidance Light):**
    1.  화면 전체에 붉은 노이즈 필터가 서서히 빠지며(Fade Out), 대신 **따뜻한 시안 블루 또는 연두색 계열의 부드러운 빛(`Guidance Light`)**이 Gap 발생 지점과 해결책 간의 경로를 따라 흐릅니다.
    2.  해결책으로 제시된 새로운 프로세스 경로는 이전의 붉은 끊김(Gap)을 **빛나는 파란색/초록색의 강력한 '연결 엣지(Strong Edge)'**로 대체하며, 이 과정이 마치 마법처럼 자연스럽게 연결되는 애니메이션을 보여줍니다.
    3.  '수정 적용 (Apply Fix)' 버튼은 단순한 버튼이 아니라, **사용자의 다음 행동을 유도하는 명확하고 입체적인 CTA 컴포넌트**로 디자인되어야 하며, 클릭 시 확실하고 만족스러운 '클릭 피드백(Click Feedback: 찰칵 소리 및 밝게 빛나는 효과)'을 제공해야 합니다.
*   **UX 피드백:** "최적화된 프로세스 경로를 통해 $Y\%$의 리스크 감소가 예상됩니다."와 같이, **긍정적인 결과 지표(Gain)**를 강조하며 사용자에게 안도감과 성취감을 동시에 부여합니다.
