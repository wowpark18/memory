## 🎓 EPHA 온보딩 시나리오 스크립트: 정보 연결 과정 탐색자 (The Flow Navigator)

**[목표]**: 사용자에게 '정보 단절'의 불편함을 공감시키고, EPHA가 단순한 데이터베이스 검색이 아닌, **프로세스적 관점에서 누락된 연결 고리($F_{Gap}$)를 찾아 해결해주는 지능형 시스템**임을 체험하게 한다.
**[톤앤매너]**: 권위적이되 안내하는 듯 친절함 (Authoritative yet Guided). 복잡한 개념을 일상적인 페인 포인트로 치환하여 설명한다.
**[러닝타임 예상]**: 2분 ~ 3분 내외

---

### 🎬 Scene 1: 문제 제기 및 공감 (The Pain Point - Gap의 발견)
*   **목표:** 사용자가 현재 겪는 '무엇이 빠졌는지 알 수 없는' 막연한 불편함에 공감하게 만든다.
*   **화면 구성**: 복잡하고 분산된 정보들(책, PDF, 웹사이트 링크 등)이 한 화면에 산개되어 마치 미로처럼 보인다.

| 구분 | 내용 (Voiceover / UI Text) | 디테일 가이드라인 |
| :--- | :--- | :--- |
| **[Hook]** | **(V/O)** "혹시, '알고는 있는데 어디서부터 시작해야 할지 막막한' 순간을 겪어보신 적 있으신가요?" | 부드러운 배경 음악과 함께 사용자 시선이 여러 정보들 사이에서 방황하는 모습을 보여준다. |
| **[Pain Point]** | **(V/O)** "우리는 수많은 정보를 모으지만, 그 정보들이 서로 어떻게 연결되어 하나의 '해결책'을 만들어내는지 알기 어렵습니다." | 화면 중앙에 커다란 물음표(?)가 깜빡인다. (정보의 단절 시각화) |
| **[Gap Trigger]** | **(UI Text)** *[🚨 경고: 핵심 프로세스 누락 감지]*<br>**(시스템 사운드: 웅장한 알림음)** | **🎨 Designer 컴포넌트 활용:** `$F_{Gap}$ Alert Box` (Warm Ivory 배경, Crimson Red 테두리)가 화면을 가로막으며 나타난다. |
| **[V/O 설명]** | **(V/O)** "이것이 바로 '정보 단절 지점'입니다. 필요한 정보는 충분한데, 그 연결 과정 자체가 비어 있는 상태죠. 이 간극을 저희는 $F_{Gap}$이라 정의합니다." | 경고창의 텍스트를 가리키며 설명한다. (전문 용어를 사용하되, '간극'이라는 단어로 풀어준다.) |

---

### 🎬 Scene 2: EPHA의 개입과 프로세스 안내 (The Edge - 해결 과정)
*   **목표:** EPHA가 감정적 위로가 아닌, **구체적인 과정을 안내하는 시스템**임을 체험하게 한다.
*   **화면 구성**: `$F_{Gap}$ Alert Box`가 나타나자마자, EPHA의 인터페이스가 부드럽게 오버레이 되며 빨간 경고를 중화시킨다.

| 구분 | 내용 (Voiceover / UI Text) | 디테일 가이드라인 |
| :--- | :--- | :--- |
| **[EPHA 등장]** | **(V/O)** "걱정 마세요. EPHA는 단순히 정보를 찾아주는 시스템이 아닙니다. 우리는 '어떻게 연결해야 할지' 그 과정을 안내합니다." | 경고창의 톤이 미묘하게 Yellow/Blue 계열로 변화하며 신뢰감을 준다. |
| **[Step 1: 분석]** | **(V/O)** "첫 번째 단계는, 현재 가지고 있는 정보들이 어떤 '규칙'과 '흐름'을 따르는지 구조적으로 파악하는 것입니다." | 화면의 산개된 정보들이 자동으로 작은 노드들로 분해되는 애니메이션이 재생된다. (분석 과정 시각화) |
| **[Step 2: 연결 지점 탐색]** | **(V/O)** "EPHA는 데이터 노드 사이를 흐르는 보이지 않는 '연결 고리'를 추적합니다. 이 경로가 바로 $F_{Gap}$을 메울 수 있는 핵심 통로입니다." | 시스템이 특정 두 노드를 연결하는 가상의 파란색 선(`Edge`)을 빛내며 그어낸다. **(Designer의 Edge 개념 활용)** |
| **[Step 3: 문제 정의 및 질문]** | **(UI Text - 상호작용 유도)** "💡 *질문:* 이 정보들이 궁극적으로 해결하려는 '최종 목적'은 무엇인가요?" | 사용자에게 능동적인 질문을 던지며, 사용자의 집중도를 높인다. (튜토리얼의 핵심 인터랙션) |

### 🎬 Scene 3: 최종 해소와 가치 확립 (The Gain - 통찰과 경험)
*   **목표:** 문제 해결 과정을 통해 얻는 '아하 모먼트'를 극적으로 보여주고, 앱 사용을 유도한다.
*   **화면 구성**: 모든 정보가 하나의 깔끔하고 논리적인 플로우차트로 재정렬되며 빛난다.

| 구분 | 내용 (Voiceover / UI Text) | 디테일 가이드라인 |
| :--- | :--- | :--- |
| **[Aha Moment]** | **(V/O)** "자, 이제 보세요. 단절되어 보이던 정보들이 하나의 명확한 '흐름'으로 완성되었습니다." | 모든 노드가 완벽하게 연결되며, 최종적인 결론을 나타내는 메인 박스가 중앙에 뜬다. (만족스러운 성공 사운드) |
| **[Core Value Recap]** | **(V/O)** "EPHA는 단순히 답을 알려주지 않습니다. 사용자가 '어떻게 생각하고, 어떤 순서로 연결해야 하는가'라는 통찰의 경로 자체를 선물합니다." | 화면에 핵심 슬로건이 등장: **"정보 단절 해소. 과정 중심의 지능형 프로세스 시스템."** |
| **[CTA]** | **(V/O)** "이제, 복잡한 정보 속에서 길을 잃지 마세요. EPHA와 함께 가장 확실하고 효율적인 연결 고리를 찾아보세요." | 명확하고 크게 '시작하기' 버튼을 노출시킨다. (버튼 애니메이션 필수) |
