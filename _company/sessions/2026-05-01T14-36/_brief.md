# 📋 작업 브리프

**원 명령:** Look-UP 프로젝트을 같이 공유 하고 싶은데 개발중이 폴더를 공유 하는 방법이나 깃허브를 통해 확인하는 방법등 가장 효과적인 방법 찾아서 가져와~!

## 요약
프로젝트 공유의 가장 효과적인 방법을 찾는 것이 목표입니다. 단순히 폴더를 공유하는 방식에서 벗어나, 개발 과정과 지식 자체가 체계적으로 공유되고 관리될 수 있는 '협업 프로세스' 관점에서 최적의 솔루션을 도출합니다.

## 분배
- **💰 Business**: 단순 폴더 공유가 아닌, 프로젝트 개발 과정(Process) 자체를 지식 자산으로 만드는 협업 철학을 정의해주세요. 우리의 목표는 '코드' 공유가 아니라, '문제 해결 방법론'의 투명한 전달이어야 합니다.
- **🔍 Researcher**: 프로젝트 공유 방법 3가지(1. 로컬 폴더 직접 공유, 2. Git 기반 리포지토리 사용, 3. 클라우드 IDE/협업 플랫폼 사용)의 장단점을 비교하는 상세 보고서를 작성해주세요. 특히 '버전 관리 용이성', '실시간 협업 가능 여부', '비즈니스 규모 확장성' 측면에서 평가가 필요합니다.
- **💻 Developer**: 연구 결과를 바탕으로 가장 효과적인 방법(Git Repository)을 채택하고, 이를 실제로 구현하기 위한 표준화된 개발 워크플로우를 제안해주세요. 'Feature Branch' 생성부터 'Pull Request (PR)' 과정을 거쳐 메인 브랜치에 병합되는(Merge) 기술적 흐름도를 상세히 작성합니다.
