## 💎 EPHA: 통합 E2E 위험 진단 플로우차트 (Risk Assessment Flowchart)

### 📌 개요 및 목표
*   **목표:** 사용자가 특정 액션(데이터 입력, 프로세스 완료 등)을 수행했을 때 시스템이 실시간으로 리스크를 분석하고($L$-Cost 산출), 그 결과에 따라 사용자에게 경고와 해결책을 제시하는 과정 정의.
*   **핵심 원칙:** '경고'는 반드시 시각적/감성적으로 압도적이어서, 사용자가 이를 가볍게 지나칠 수 없도록 설계해야 합니다.

### 🔄 플로우차트 상세 구조 (Flowchart Detail)

| 단계 | 사용자 액션/시스템 트리거 | 시스템 동작 (API 호출) | API 응답 분석 및 UI 반응 디렉션 |
| :--- | :--- | :--- | :--- |
| **START** | **[사용자 입력]** 사용자가 필수 정보를 입력하고 '검증(Validate)' 버튼 클릭. <br/>*(예: 프로젝트 스코프 확정, 데이터 연결 완료)* | ➡️ `POST /api/v1/risk-assessment` 호출 (Workflow ID 포함) | **[Loading State]** 화면 중앙에 미니멀한 그리드 패턴과 함께 '지식 연결 분석 중...' 애니메이션 표시. (딥테크 신뢰도 강조). 로딩 시간을 최소화하고, 계산이 깊고 복잡하다는 느낌을 줘야 함. |
| **DECISION** | **[시스템 처리]** 백엔드가 `risk_status`를 반환. | ➡️ JSON 응답 수신 및 파싱. | **IF/ELSE 구조 기반의 비주얼 분기.** (아래 세 가지 시나리오로 분리) |

### 🔴 시나리오 A: High Risk (위험 발생 - 재무적 위협 감지)
*(Mock API 예시: level="High", l_cost=N, warning_code="CRITICAL-XXX")*

1.  **비주얼 전환:** **강렬한 시스템 경고 모드 진입.** 배경 전체에 어두운 톤(Deep Black/Navy #0A192F)의 필터가 씌워지고, 주변 UI 요소들이 희미하게 처리됨 (Dimming Effect).
2.  **메시지:** 화면 중앙 상단에 **`🚨 CRITICAL ALERT: Process Integrity Breach Detected!`** 경고 배너를 붉은색(Crimson Red #B03A2E)으로 전면 배치.
3.  **정보 시각화:** API의 `l_cost` 값을 가장 크고, 깜빡이는 애니메이션(`Pulse Effect`)과 함께 표시합니다. **"예상 손실액: ₩[L-Cost] (가장 큰 폰트)"**.
4.  **UX/Interaction:** 화면이 일시적으로 멈추는 듯한(Halt) 효과를 주어 사용자의 시선 강제 고정. `details`의 `action_required` 항목들을 **'Must Act' 카드 컴포넌트** 형태로 팝업하고, 각 액션 옆에 '필수', '지연 시 비용 증가' 등의 레이블을 붙여 위기감을 극대화합니다.
5.  **Next Step:** 사용자가 경고 메시지를 확인(클릭)해야만 다음 단계로 진행 가능하게 설계 (Hard Stop).

***

### 🟡 시나리오 B: Medium Risk (주의 및 Gap 발생 - 개선 필요 지점 감지)
*(Mock API 예시: level="Medium", l_cost=M, warning_code="GAP-XXX")*

1.  **비주얼 전환:** **경고 안내 모드 진입.** 배경에 주황색(Amber Orange #FFC300)의 은은한 그라데이션 오버레이를 적용합니다. (High Risk보다는 덜 위협적이지만, 무시할 수 없는 경고).
2.  **메시지:** 화면 상단에 **`⚠️ ATTENTION: Information Gap Detected.`** 메시지를 배치합니다.
3.  **정보 시각화:** API의 `details`에서 감지된 구체적인 단절 지점(`gap_detected`)을 인터랙티브하게 하이라이트(Highlight) 합니다. (예: 플로우차트 상의 특정 노드-엣지 연결 부분이 깜빡임).
4.  **UX/Interaction:** 'Gap 유형'과 '잠재적 영향도($L$-Cost)'를 비교하는 **'Before/After 카드 컴포넌트'**를 제시합니다. 사용자가 이 간극(Gap)을 인지하고, `action_required` 중 가장 쉬운 액션부터 취하도록 유도하며, 가이드 역할을 수행합니다.
5.  **Next Step:** 경고 수락 및 필요한 데이터/정보 입력 후 다음 단계로 자연스럽게 진행 (Soft Stop).

***

### 🟢 시나리오 C: Low Risk (정상 프로세스 - 무결성 확보)
*(Mock API 예시: level="Low", l_cost=L, warning_code="OK-000")*

1.  **비주얼 전환:** **안심 및 확신 모드 진입.** 배경의 전반적인 톤을 깨끗한 밝은 색상(Off-White #F7F9FA)으로 유지하고, 성공 요소에만 시안 블루(#4A90E2) 계열로 포인트를 줍니다.
2.  **메시지:** 화면 상단에 **`✅ Process Integrity Confirmed.`** 메시지를 배치합니다. (성공이 아니라 '보험'으로서의 무결성을 확인했다는 뉘앙스 유지).
3.  **정보 시각화:** API가 계산한 `l_cost`를 보여주되, 이는 '현재 리스크 비용'이 아닌 **'확보된 안정성 가치(Assurance Value)'**로 재해석하여 표시합니다.
4.  **UX/Interaction:** 사용자에게 다음 단계의 성공적인 결과물에 대한 기대감을 높이는 메시지("이제 핵심 과제인 [다음 목표]를 수행할 준비가 되었습니다.")와 함께, 시스템이 자체적으로 제공하는 **'최적 경로 제안(Optimal Path Suggestion)'** 컴포넌트를 노출합니다.
5.  **Next Step:** 별도의 방해 없이 다음 프로세스로 매끄럽게 진행 (Seamless Transition).
