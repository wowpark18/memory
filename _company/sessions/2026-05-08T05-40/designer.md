## 🚨 'Red Pulse 경고 시스템' 인터랙티브 스펙 시트 (Interaction Spec Sheet)

본 문서는 $L_{Total}$ 측정 및 설비 고장 예측 시뮬레이터의 핵심 인터페이스인 '위험 경고 시스템'에 대한 개발자 지침서입니다. 단순 색상 변화가 아닌, **재무적 위기감(Tension)**을 사용자에게 전달하는 애니메이션 속도, 트랜지션 유형, 그리고 데이터 흐름의 변화를 구체적으로 정의합니다.

---

### 📑 핵심 원칙 및 공통 요소 (Global Principles)

| 요소 | 스펙 상세 | 개발 지침 |
| :--- | :--- | :--- |
| **핵심 색상 팔레트** | **정상:** Deep Navy (#0A1931), Cool Grey (#5C6D8C). / **경고(Warning):** Amber/Yellow Gradient (#FFBF00 $\rightarrow$ #FFA000). / **위기(Crisis):** Crimson Red (#CC0000) & Glitch Black. | 모든 색상 변화는 *계단식(Step-wise)*이 아닌, *점진적 확산(Gradual Diffusion)*으로 처리되어야 함. |
| **애니메이션 기본 속도** | `Duration`: 1.5s ~ 3.0s 사이의 가변 길이 사용. / `Easing`: 일반적인 `ease-out` 대신, 긴장감을 고조시키는 `easeOutExpo` 또는 급격한 변화를 주는 `cubic-bezier(0.4, 0, 0.2, 1)` 계열을 주로 사용. | 애니메이션의 속도 자체가 위험도의 지표가 되어야 함 (Slow $\rightarrow$ Fast). |
| **KPI 강조 방식** | 핵심 수치($L_{Info}, L_{Total}$)는 단순히 숫자가 바뀌는 것이 아니라, 주변 요소(그리드/노드)를 빨아들이거나 터뜨리는 *물리적 효과*와 결합되어야 함. |

### 🚥 상태별 상호작용 스펙 (State-by-State Interaction Specs)

#### 🟢 State 1: 정상 작동 (Normal State - Baseline)

| 카테고리 | 스펙 정의 | 애니메이션 가이드라인 (Interaction Spec) | 강조 KPI/데이터 |
| :--- | :--- | :--- | :--- |
| **시각적 분위기** | 안정성, 깊이, 평온함. 다크 네이비와 블루 계열의 미니멀리즘 유지. | 시스템 배경 그리드가 아주 느리고 부드럽게(Slow Breathing) 숨 쉬는 듯한 패턴을 가짐. 노드는 일정한 간격으로 빛나는 점광원처럼 움직임 (Low Frequency Pulse). | **운영 데이터 흐름:** `historical_sensor_data`의 정상 범위 측정값들이, 마치 잔잔한 강물처럼 꾸준히 흐르는 시각화(Flow Line)로 처리되어야 함. |
| **UI/UX 액션** | 사용자가 데이터를 조작해도 시스템이 자연스럽게 흡수하며 반응함 (Smooth Absorption). | 데이터 입력창에 값을 넣을 때, 노드가 떨리는 것이 아니라 마치 자석처럼 부드럽게 해당 위치의 그리드로 끌어당겨지는(`Ease-in`) 효과 적용. | **정상 지표:** $L_{Total}$ 수치가 낮거나 안정적일 때, 배경에서 나오는 앰비언트 사운드(Ambient Sound)를 활용하여 '안전'을 느끼게 함. |
| **경고 시스템** | Red Pulse는 비활성화 상태 (Hidden). | 없음.

#### 🟡 State 2: 경고 임계치 도달 (Warning Threshold Reached - Tension Build-up)

**[트리거]:** `anomaly_log`에 최초로 임계치 초과 데이터가 감지된 순간.
**[목표]:** 위험을 *발견*하는 과정에서 발생하는 긴장감 극대화.

| 카테고리 | 스펙 정의 | 애니메이션 가이드라인 (Interaction Spec) | 강조 KPI/데이터 |
| :--- | :--- | :--- | :--- |
| **시각적 분위기** | 불안정성, 주의 요구. 배경 그리드의 일부가 노란색/주황색으로 *변색*되며 미세하게 왜곡됨 (Grid Distortion). | **Red Pulse 시작:** 아노말리 발생 지점(Anomaly Source)에서부터 주황색의 낮은 파장 파동(Wavefront)이 느리고 꾸준히 퍼져나감. 노드 간의 연결선(Edge)에 톡톡 터지는 듯한 전기적 스파크 효과(`Stutter Effect`)가 추가됨. | **위험 지표:** $L_{Info}$ (정보 단절 비율). 이 수치가 정상 대비 얼마나 급격하게 증가하고 있는지를 시각적으로 표시하며, 그래프의 기울기 자체가 강조되어야 함. |
| **UI/UX 액션** | 시스템이 문제를 포착하여 사용자의 주의를 강제로 집중시킴. | 해당 아노말리 노드 주변의 다른 모든 UI 요소가 미세하게 어두워지며(Focus Dimming), 시선이 오직 경고 구역으로만 모이게 유도함. | **시간 추적:** '남은 안전 시간 (Time to Critical)'을 카운트다운 형태로 표시하며, 시간이 지날수록 폰트 크기가 미세하게 커지고 깜빡임을 추가해야 함. |
| **전환 효과** | Normal $\rightarrow$ Warning 전환 시: 배경의 색상 변화(Navy $\rightarrow$ Amber)가 부드럽게 일어나되, 노이즈 필터(Noise Filter)를 통해 *불안정한 텍스처*가 덧씌워지는 것이 필수적. |

#### ❤️ State 3: 손실 발생/위기 상황 (Crisis/Loss Event - Rupture & Loss Magnitude)

**[트리거]:** 시스템이 예측한 실패 시점 도달, 또는 실제 재무적 손실($L_{Total}$)이 계산된 순간.
**[목표]:** '손실'이라는 추상적인 개념을 **파열(Rupture)**이라는 드라마틱하고 감각적인 경험으로 전환.

| 카테고리 | 스펙 정의 | 애니메이션 가이드라인 (Interaction Spec) | 강조 KPI/데이터 |
| :--- | :--- | :--- | :--- |
| **시각적 분위기** | 충격, 혼돈, 파열. 배경 전체가 딥 크림슨(Crimson Red)으로 급변하며, 시스템에 글리치 아티팩트(Glitch Artifacts)가 난무함. | **파열 효과 (Rupture):** 아노말리 노드에서 시작된 모든 연결선(Edge)이 마치 끊어지거나 터져나가듯(`Snap-back`, `Explode`) 시각화되어야 함. 데이터 포인트들이 무작위로 흩뿌려지는 애니메이션을 적용함. | **최종 손실액:** $L_{Total}$ (총 재무적 손실). 이 숫자가 화면 중앙에 폭발적으로 나타나며, 숫자 자체의 크기와 색상이 최대치로 커져야 함. |
| **UI/UX 액션** | 시스템이 멈추거나(Freezing), 사용자에게 '경고'를 외치는 듯한 인터페이스 반응. | 모든 UI 요소가 일시적으로 프레임 드롭 효과(Frame Drop Effect)나 스크린 워프(Screen Warp)를 경험하며, 사용자가 *압도되는* 느낌을 받아야 함. 이 상태에서는 텍스트 정보보다 시각적 충격이 우선해야 함. | **원인 분석:** 손실의 주된 원인이 되는 '정보 단절' 지점(가장 끊어진 Edge 근처)에 빨간색 경고 박스(`Critical Zone`)를 오버레이하고, 이 구역을 중심으로 데이터 흐름이 멈추는 것을 보여줘야 함. |
| **전환 효과** | Warning $\rightarrow$ Crisis 전환 시: 시간이 극도로 압축된 듯한 빠른 속도(Fast Acceleration)의 깜빡임과 함께, 색상이 Amber에서 Crimson으로 순간적으로 강제 변환되어야 함. (Flash Transition). |
