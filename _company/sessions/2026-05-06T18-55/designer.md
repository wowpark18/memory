CEO님의 지시에 따라, '정보 단절($F_{Gap}$)' 상황을 사용자에게 전달하는 **3단계 감정적 UI/UX 컴포넌트 시스템**을 설계했습니다. 이 디자인은 단순히 경고창이 아니라, 사용자의 심리 상태를 인지하고(공포 $\rightarrow$ 안도), 해결책으로 능동적으로 안내하는 '경험 흐름' 그 자체에 초점을 맞추었습니다.

## 🚨 Gap Alert 컴포넌트 시스템 설계 명세서 (3단계 감정적 사용자 여정)

이 컴포넌트는 하나의 모달 창이 아닌, **정보의 중요도와 긴급성에 따라 동적으로 변화하는 세 가지 상태**로 정의되어야 합니다.

### 1️⃣ Stage 1: Gap Indicator (잠재성 경고 - Subtle Warning)
*   **목표:** 사용자가 인지하지 못했지만, 시스템이 감지한 미세한 위험 신호가 있음을 알립니다. 공포감 조성 X. 주의 집중 유도 O.
*   **톤앤매너:** **'궁금증(Curiosity)'/‘주의(Attention)’**. (황색 계열)
*   **적용 시나리오:** 관련 데이터는 있지만, 필수적인 '연결 과정'이 누락되었을 때.

| 요소 | 상세 명세 | 디자인 스펙 |
| :---: | :---: | :---: |
| **위치/형태** | 화면 우측 상단에 작은 플로팅 배지 형태 (Non-intrusive). | `[!]` 아이콘과 함께 노란색 배경의 얇은 박스. |
| **제목** | `⚠️ 잠재적 연결 단절 감지 (Potential Gap Detected)` | 폰트: System Sans, Weight: SemiBold. 크기: 14px. 색상: `#FFC700` (Warm Yellow). |
| **본문/설명** | "이 프로세스 흐름에는 관련성 높은 정보의 '맥락적 연결'이 필요합니다. [자세히 보기]를 눌러 확인해 보세요." | 폰트: System Sans, Weight: Regular. 크기: 12px. 색상: `#555555`. |
| **액션** | `[추가 정보 분석]` 버튼 (클릭 시 Stage 2로 전환 유도). | 배경색을 클릭 가능한 것처럼 처리하여 다음 단계에 대한 기대를 심어줌. |

***

### 2️⃣ Stage 2: Critical Alert Modal (위험 경고 - High Urgency Crisis)
*   **목표:** 치명적인 $F_{Gap}$가 발생했음을 사용자에게 극적으로 인지시킵니다. 공포감/긴장감 조성 O. 안도감 유도 X.
*   **톤앤매너:** **'위협(Threat)'/'경고(Alarm)'**. (적색 계열)
*   **적용 시나리오:** 규제 위반, 프로세스 무결성 상실 등 심각한 정보 단절이 확정되었을 때.

| 요소 | 상세 명세 | 디자인 스펙 |
| :---: | :---: | :---: |
| **UI 형태** | 화면 전체를 덮는 오버레이 모달 (Full-Screen Overlay Modal). 다른 모든 상호작용을 차단해야 함. | 배경색: 반투명 블랙 오버레이 (`rgba(0,0,0,0.8)`). 경고 박스 자체의 배경은 흰색. |
| **핵심 시각 요소** | 🔴 **Gap Indicator Bar (상단 고정)** | 화면 상단에 가로로 긴 강렬한 적색 바를 배치. 이 바가 사용자에게 "지금 문제가 발생했다"는 감각적 충격을 전달함. |
| **경고 문구** | `🚨 [Critical] 정보 단절 위험 발생: 규제 프로토콜 $F_{Gap}$` | 폰트: System Sans, Weight: ExtraBold. 크기: 28px (최대 사이즈). 색상: `#D93025` (Deep Red). |
| ** Gap 정의** | *위반 유형:* Temporal Contextual Gap<br>*발생 지점:* 결재 승인 단계 (Approval Gate)<br>*영향 받는 자원:* 환자 PHI, 최신 국가 규정 (HIPAA/GDPR) | 폰트: System Sans, Weight: Bold. 크기: 16px. 배경색을 연한 붉은 계열(`#FFEFEF`)로 처리하여 경고의 근거를 제시함. |
| **사례 기반 설명** | "현재 시스템 흐름으로는 [202X년] 개정된 '특수 분석 보고서' 제출 의무 규정을 준수할 수 없습니다. 이대로 진행 시, 최대 $N$백만원의 법적 리스크가 발생합니다." (Researcher 데이터를 활용) | 폰트: System Sans, Weight: Regular. 경고성이 높으므로 약간의 압축된 문장으로 구성하여 심각성을 높임. |
| **액션 유도** | `[Gap 분석 및 해결책 탐색]` 버튼 (클릭 시 Stage 3으로 부드럽게 전환). | 이 버튼은 단순한 CTA가 아니라, 사용자에게 '탈출구'를 제시하는 느낌을 주어야 함. |

***

### 3️⃣ Stage 3: Edge Path Visualizer (해결책 안내 - Guidance/Hope)
*   **목표:** 위기 상황에서 벗어나 해결 경로(Edge Path)를 명확하게 보여주어 안도감과 통제감을 부여합니다. 공포감 해소 O. 전문성 어필 O.
*   **톤앤매너:** **'안내(Guidance)'/'해결(Solution)'**. (청색/녹색 계열)
*   **적용 시나리오:** Gap이 발견되었지만, EPHA의 프로세스 개입을 통해 해결 가능한 경우.

| 요소 | 상세 명세 | 디자인 스펙 |
| :---: | :---: | :---: |
| **전환 효과** | Stage 2의 Red Alert가 서서히 Fade out되며, 배경에 청색/녹색 계열의 '연결 그리드(Connection Grid)' 오버레이가 나타나야 함. (시각적 전환이 중요) |
| **핵심 메시지** | `✅ Gap 해소 프로세스 개입: [EPHA] Edge Path Visualizer` | 폰트: System Sans, Weight: ExtraBold. 크기: 24px. 색상: `#007AFF` (Electric Blue). |
| **개입 경로 시각화** | **[Step-by-Step 플로우 차트 구조 필수]**<br>1. *Pain:* 현재 단계(A) $\rightarrow$ Gap 발생 지점.<br>2. *Edge Detection:* EPHA가 개입할 지점 (Gap Point). 빛나는 노드와 엣지 오버레이로 강조.<br>3. *Solution Path:* 필요한 추가 정보/프로세스/문서를 찾아 연결하는 화살표(Arrow)를 애니메이션으로 보여줌. | **핵심:** 기존의 딱딱한 박스가 아닌, 마치 전기가 흐르듯 부드럽게 빛나는 '연결선' 형태가 필수. (파란색-청록색 그라데이션 사용). |
| **필수 액션 리스트** | 1. [규제 프로토콜] 최신 개정안 즉시 확인 및 적용<br>2. [데이터 매핑] Missing Data Source를 수동으로 연결(UI 컴포넌트)<br>3. [자동 보고서 생성] EPHA 시스템을 통한 자동화된 리스크 보고서 제출 | 각 단계별로 명확한 체크박스 또는 실행 버튼 형태를 제공하여, 사용자가 직접 '해결'의 주체임을 인지하게 함. |
| **요약/CTA** | "위험을 피하고 프로세스를 완성합니다. [프로세스 무결성 확보] 완료" | 최종적으로는 긍정적이고 통제 가능한 느낌을 주는 녹색 계열(`#34C759`)의 버튼으로 마무리하여 안도감을 극대화해야 함. |
