# 📄 EPHA MVP 버전 1.0 핵심 기능 정의서 (Feature Definition Document)
## $L_{Total}$ 기반 '정보 단절 위기감' 체험 서비스

이 문서는 정보의 비효율적 흐름에서 발생하는 **측정 가능한 재무적 잠재 손실($L_{Total}$)을 사용자에게 진단하고, 해결책을 판매**하는 MVP 1.0 버전을 위한 구체적인 스펙 및 수익화 전략을 정의합니다.

---

### 1. 🎯 서비스 모델 (Service Model & Value Proposition)

| 구분 | 내용 | 핵심 가치 제안 (Value Prop) |
| :--- | :--- | :--- |
| **서비스명** | EPHA $L_{Total}$ Gap 진단기 | "당신이 모르는 사이에 발생하는 잠재적 손실, 숫자로 확인하세요." |
| **핵심 기능** | **Gap Analysis Report (정보 단절 리포트)**: 사용자가 제공한 지식/데이터 흐름을 분석하여 '최적 경로와 실제 경로 간의 괴리(Gap)'를 시각화합니다. 이 Gap이 곧 재무적 위험 $L_{Total}$로 측정됩니다. | 단순 정보 전달이 아닌, **'위험 진단 경험'** 자체를 판매하는 서사적 접근입니다. (판매 포인트: 문제 정의) |
| **수익화 흐름** | 1. 사용자가 무료/저가 티어로 **Gap Score 및 Red Pulse 경고** 체험 $\rightarrow$ 2. $L_{Total}$의 심각성을 체감하고, 더 깊은 분석을 원함 (Pain Point 극대화) $\rightarrow$ 3. 유료 리포트(Tier 2 이상) 구매 또는 구독 결제 → 4. EPHA가 제공하는 **'정보 연결 솔루션/컨설팅 연계'**로 전환. |
| **MVP 핵심 워크플로우 (1단계)** | 사용자 입력 $\rightarrow$ 그래프 분석 $\rightarrow$ $L_{Total}$ 점수 산출 및 경고색(Red Pulse) 표시 $\rightarrow$ **"더 자세한 원인 분석은 유료 리포트를 통해 확인하세요."**라는 CTA 제시. |

### 2. 💰 수익화 가격 정책 예시 (Tiered Pricing Model)

MVP의 목표는 '체험을 통한 위기감 고조'이므로, 가장 낮은 티어부터 높은 티어까지 점진적으로 '해결책에 대한 갈망'을 키워야 합니다.

| Tier | 상품명 및 포지셔닝 | 포함 기능 (Scope) | 가격 정책 예시 |
| :--- | :--- | :--- | :--- |
| **FREE** | **$L_{Total}$ Quick Scan (진단 맛보기)** | 1회성 기본 Gap Score 산출. 가장 큰 위험 노드(Red Pulse) 3개만 시각화 제공. 제한적 데이터 입력량. | ₩0 (고객 확보용 Hook) |
| **PREMIUM** | **$L_{Total}$ Deep Dive Report (월간 구독)** | 무제한 Gap 분석 및 상세 리포트 다운로드. 시간 흐름(Time Stream) 기반의 인과관계 추적 기능 제공. 맞춤형 경고 알림 설정. | 월 ₩29,000 (주요 매출원/지속 가능성 확보 목적) |
| **ENTERPRISE** | **Consulting Linkage & API Access (맞춤 컨설팅)** | 기업 데이터 연동 및 분석. 전담 컨설턴트 연결(EPHA 내부 또는 제휴사). Gap 발생 시 자동화된 해결 로직 추천서 제공. | 별도 견적 / 프로젝트 기반 (High-Ticket Item) |

### 3. ✨ 핵심 성과 지표 (KPI: Key Performance Indicators)

MVP의 성공은 '사용자가 우리 서비스를 통해 충분한 위기감을 느끼고, 이를 해소하기 위해 돈을 쓸 의향이 생겼는지'로 측정되어야 합니다.

| KPI 항목 | 정의 및 목표 설정 근거 | 측정 방법 / 가설적 목표치 (T-1 기준) |
| :--- | :--- | :--- |
| **KPI 1: 평균 연결 경로 추적 깊이** | 사용자가 무료/저가 티어에서 Gap 분석 과정의 노드(Node)를 탐색하는 평균 단계 수. 이 숫자가 높을수록 '문제 인식'에 성공했음을 의미합니다. | **목표:** 사용자당 평균 4개 이상의 정보 연결 경로 시각화 확인 (Baseline 대비 +30%). |
| **KPI 2: 유료 전환율 (Conversion Rate)** | Gap 분석 보고서를 본 사용자가 Premium 티어 결제 페이지까지 도달하여 실제로 결제하는 비율. (진단 경험 $\rightarrow$ 구매 결정) | **목표:** Free to Paid Conversion Rate 5% 달성. (이는 제품의 가치가 돈으로 측정 가능함을 입증함.) |
| **KPI 3: 재방문 위험 모니터링 빈도** | 사용자가 서비스를 이탈한 후, 일정 기간(예: 7일) 뒤에 다시 접속하여 '새로운 리스크 점검'을 시도하는 비율. (지속적인 가치 제공 입증) | **목표:** 주간 활성 사용자 대비 재방문율(WAA/WAU) 20% 달성. (서비스가 일회성이 아닌 '필수적 체크 도구'로 인식되어야 함.) |
