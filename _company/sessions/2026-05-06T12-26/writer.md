## 💡 CEO 지시 수행: MVP 핵심 사용자 여정(User Journey Script) 초안

제공된 3가지 $F_{Gap}$ 사례를 기반으로, 사용자가 **'불편함/위험 인식 $\rightarrow$ 시스템 입력 시도 $\rightarrow$ EPHA의 프로세스 개입 및 손실 완화 경험'**을 할 수 있도록 대화형 스크립트를 작성했습니다.

이 스크립트는 단일 영상 또는 인터랙티브 데모 형태로 구성되어야 하며, 특히 **[시스템 피드백]** 부분에서 에이전트가 '지식 전달'이 아닌 '프로세스 안내 및 경고'에 초점을 맞추는 것이 핵심입니다.

### 🎬 전체 구조 개요 (데모 흐름)
*   **인트로:** "당신의 데이터가 고립되어 있습니다. 이 간극(Gap)이 돈과 시간을 앗아갑니다." (문제 제기)
*   **본론:** 3가지 Case를 순차적으로 시연하며, EPHA가 개입하는 과정을 보여줍니다.
*   **클로징:** "EPHA는 데이터를 연결하는 것이 아니라, *프로세스를 복구*합니다." (솔루션 제시 및 CTA)

### 🧩 Part 1: AML/KYC 프로세스 Gap 시나리오 (위험 감지 & 정보 연결)

**[상황 설정]**: 거래 모니터링 팀이 평소와 다른 의심스러운 대규모 자금 이체 패턴을 발견했습니다. 하지만 초기 고객 확인(KYC) 단계의 정보가 부족하여, 이 패턴이 정상적인 것인지 확실히 판단할 수 없습니다.
**[목표]**: 데이터 사일로를 강제로 연결하여 잠재적 리스크를 식별하는 과정 시각화.

| 단계 | 사용자 행동 (User Action) | 시스템/EPHA 반응 (System Feedback / Dialogue Flow) | 핵심 학습 포인트 |
| :--- | :--- | :--- | :--- |
| **1-Gap 인식** | *사용자(분석가)*: 의심스러운 거래 ID XXXXX를 입력하고 분석을 요청합니다. | **[시스템 경고/Red]**: "🚨 **경고:** 해당 거래의 잠재적 리스크 점수는 높으나, 관련 고객 온보딩 이력이 단편적입니다. *정보 출처 Gap*으로 인해 오판 가능성이 있습니다." | 문제가 '데이터 부족'이 아닌 '프로세스 연결 실패'임을 인식시킴. |
| **2-데이터 입력** | *사용자*: 시스템에 수동으로 추가적인 고객 정보를 (예: 과거 주소 변경 기록, 다른 계좌의 활동 기록) 업로드하려고 시도합니다. | **[시스템 오류/Red]**: "❌ **오류:** 이 데이터는 현재 거래 모니터링 프로세스 플로우와 직접 연결되지 않아 검증할 수 없습니다. *데이터 노드가 단절*되어 있습니다." | 단순히 데이터를 넣는다고 해결되지 않음을 체험하게 함. |
| **3-EPHA 개입** | *사용자*: EPHA의 'Gap 분석' 버튼을 클릭합니다. | **[시스템 프로세스/Blue]**: "🔄 **EPHA 작동 시작:** 주변 데이터 노드(KYC 기록, 과거 거래 스트림)를 재조합하여 새로운 연결 경로를 탐색 중..." (시각화 플로우 애니메이션). | EPHA가 *어디*에서 정보를 가져와서 어떻게 연결하는지 보여줌. |
| **4-손실 완화** | (잠시 후) | **[시스템 성공/Green]**: "✅ **리스크 재평가 완료:** Gap 분석을 통해 고객이 6개월 전 '다른 목적'으로 유사한 패턴의 활동을 했음이 확인되었습니다. *프로세스 흐름*상 이를 의심 거래로 분류합니다. (잠재적 벌금 리스크 $F_{Gap}$ 감소)." | 단순 지식 전달이 아니라, **'과거와 현재를 연결하여 위험을 사전에 제거하는 과정'**의 가치 증명. |

### 🧩 Part 2: 개인정보 거버넌스 Gap 시나리오 (규제 준수 & 추적성)

**[상황 설정]**: 고객이 "GDPR에 따라 모든 민감 정보 삭제를 요청"했습니다. 회사는 여러 레거시 시스템에 PII가 분산되어 있어, 어디서 데이터를 지워야 할지 알 수 없습니다.
**[목표]**: 데이터 라이프사이클 전반을 추적하여 누락되는 '정보의 틈(Gap)'을 찾아내는 과정 시각화.

| 단계 | 사용자 행동 (User Action) | 시스템/EPHA 반응 (System Feedback / Dialogue Flow) | 핵심 학습 포인트 |
| :--- | :--- | :--- | :--- |
| **1-Gap 인식** | *사용자(담당자)*: '데이터 삭제 요청'을 접수하고, 내부 데이터베이스 3곳에 개별적으로 문의합니다. | **[시스템 경고/Yellow]**: "⚠️ **주의:** A 시스템에서는 PII를 찾았으나, C 마케팅 DB의 로그 기록에는 해당 고객 ID가 남아있어 삭제 여부를 확인할 수 없습니다. *정보 흐름 Gap*이 존재합니다." | '수동 확인'으로는 규제 준수가 불가능함을 깨닫게 함. |
| **2-데이터 입력** | *사용자*: EPHA 시스템에 고객의 요청 정보(ID, 삭제 범위)를 최초 입력합니다. | **[시스템 프로세스/Blue]**: "🔄 **EPHA 작동 시작:** 중앙 메타데이터 레지스트리를 기반으로 PII가 잠재적으로 존재하는 모든 모듈을 매핑 중..." (전사 데이터 흐름도 플로우 차트 애니메이션). | EPHA가 전체 시스템의 지도를 그린다는 개념 제시. |
| **3-Gap 발견** | (잠시 후) | **[시스템 경고/Red]**: "🚨 **긴급 Gap 발견:** '구(舊) 보고서 아카이브' 모듈에서 해당 고객 ID에 연결된 PII가 삭제되지 않은 것으로 확인되었습니다. 이는 규제 위반 리스크를 야기할 수 있습니다." (위치와 이유 명시). | 단순히 '지운다'는 개념이 아니라, **'어디에 남아있을지 예상하고 찾는다'**는 능동적 프로세스 제시. |
| **4-손실 완화** | *사용자*: EPHA가 지정한 자동 삭제 워크플로우를 실행합니다. | **[시스템 성공/Green]**: "✅ **준수 완료:** PII의 수집, 사용, 삭제 과정 전체에 대한 감사(Audit) 로그가 자동으로 기록되었습니다. 법규 준수 리스크 $F_{Gap}$ 완화." | 프로세스를 통해 규제 리스크를 사후 대응이 아닌 사전 예방으로 관리하는 가치 제시. |

### 🧩 Part 3: 글로벌 규제 리포팅 Gap 시나리오 (표준화 및 통합)

**[상황 설정]**: 다국적 기업의 재무팀이 여러 국가(미국 GAAP, 유럽 IFRS 등)에서 수집된 매출 데이터를 한 곳에 모아 최종 보고서를 작성해야 합니다. 기준 통일이 어렵습니다.
**[목표]**: 이질적인 원천 지식을 EPHA의 '통합 프로세스'를 통해 표준화하고 하나의 의미 있는 결과로 도출하는 과정 시각화.

| 단계 | 사용자 행동 (User Action) | 시스템/EPHA 반응 (System Feedback / Dialogue Flow) | 핵심 학습 포인트 |
| :--- | :--- | :--- | :--- |
| **1-Gap 인식** | *사용자(재무 분석가)*: 국가별로 받은 원본 매출 데이터 3세트(CSV 파일)를 업로드합니다. | **[시스템 경고/Yellow]**: "⚠️ **주의:** 각 지역 보고서는 상이한 회계 기준(GAAP vs IFRS)을 사용하고 있어, 단순 합산 시 재무적으로 왜곡된 수치($F_{Gap}$)가 발생할 위험이 높습니다." | 문제가 '데이터 양'의 문제가 아니라 '기준 체계'의 차이임을 강조. |
| **2-시스템 입력** | *사용자*: EPHA 시스템에 여러 원본 데이터를 동시에 업로드합니다. | **[시스템 프로세스/Blue]**: "🔄 **EPHA 작동 시작:** 지역별 회계 기준 매핑 레이어(Adapter Layer)를 활성화하여 핵심 재무 지표들을 추출 및 표준화 중..." (다양한 언어/기준의 데이터가 하나의 플로우로 빨려 들어가는 애니메이션). | EPHA가 '데이터 변환기' 역할을 한다는 점을 직관적으로 보여줌. |
| **3-Gap 해결** | (잠시 후) | **[시스템 프로세스/Blue]**: "🔎 **갭 분석 진행 중:** 미국 기준의 '운영 비용'과 유럽 기준의 '일반 관리비'는 개념적 차이가 있어, 표준화에 변환 계수(Conversion Factor)가 필요합니다." | EPHA가 단순히 합산하는 것이 아니라, **근본적인 프로세스 논리**를 적용함을 보여줌. |
| **4-손실 완화** | (최종 결과물 제시) | **[시스템 성공/Green]**: "✅ **통합 보고서 생성 완료:** 표준화된 '전사 통합 매출 지표(EPHA Standard)'를 기준으로 최종 리포팅을 완료했습니다. 지역별 기준 차이로 인한 재무적 오판($F_{Gap}$) 위험을 제거했습니다." | 복잡하고 어려운 업무도 EPHA의 프로세스만 거치면 명확한 하나의 통찰로 변환됨을 증명. |
