## 📚 EPHA 지식 흐름(Knowledge Flow) 시각 디자인 시스템 구축 보고서

CEO님의 지시에 따라, Writer가 작성할 모든 교육 콘텐츠 시리즈에 일관되게 적용될 **'정보 단절 해소 프로세스 (Edge)'** 기반의 비주얼 디자인 시스템을 설계했습니다. 이 시스템은 복잡한 개념적 연결(Conceptual Connection)과 정량적 리스크 제거 과정을 시각적으로 극대화하는 데 초점을 맞춥니다.

### 1. ✨ 핵심 브랜드 요소 정의 (Visual System Core)

#### A. 컬러 팔레트 및 역할 정의
| 색상 이름 | HEX Code | 용도/의미 | 적용 예시 |
| :---: | :---: | :---: | :---: |
| **Deep Navy** (기반색) | `#0A1931` | 배경, 신뢰성, 깊이. 콘텐츠 전체를 아우르는 권위적인 톤 유지. | 섹션 헤더 배경, 주 타이포그래피 컬러. |
| **Accent Blue** (연결성) | `#4A90E2` | 정보의 흐름(Edge), 연결 프로세스 자체. EPHA가 개입하는 '액션'을 상징. | 화살표, 플로우 라인, 핵심 개념 박스 테두리. |
| **Warning Red** (위험/Gap) | `#D9534F` | 정보 단절 영역($F_{Gap}$), 위협, 미확인 리스크. 문제가 발생하는 '지점'을 강조. | $F_{Gap}$ 표시 구간의 노드 배경, 경고 문구 하이라이트. |
| **Success Green** (해소/Mitigation) | `#5CB85C` | 문제 해결, 회복(Recovery), 정량적 이득. 프로세스를 통해 얻는 '안전'과 '성공'을 상징. | 최종 결과물 박스 배경, 성공적인 경로를 나타내는 강조색. |
| **Neutral Gray** (데이터/정보) | `#E0E0E0` | 일반 데이터, 기본 정보, 구조적 요소. | 단순 정보 노드(Node)의 배경 또는 구분선. |

#### B. 타이포그래피 시스템 정의
*   **헤딩 (H1-H3):** Pretendard Bold / 28px ~ 24px (굵고 명료하게 주제를 각인)
*   **본문 (Body Text):** Pretendard Regular / 16px (가독성이 높고 학술적인 깊이를 유지)
*   **강조/용어:** Monospace Font (예: JetBrains Mono) / 시스템 로그나 데이터 코드, $F_{Gap}$ 변수 등 **기술적 전문성**을 강조할 때 사용.

### 2. 🧬 지식 흐름 다이어그램 구성 요소 (The Core Diagram Set)

이 시스템의 핵심은 '선형적인 시간 축'과 '정보의 연결 경로(Edge)'를 중심으로 설계되었습니다. 모든 복잡한 과정을 이 세 가지 기본 컴포넌트를 조합하여 표현합니다.

#### A. [START] 정보 노드 (Information Source Node)
*   **기능:** 문제 발생의 원인이 되는 단편적 데이터나 기존 지식을 나타냅니다.
*   **디자인:** 직사각형 또는 불규칙한 구름 모양 박스. 배경색은 **Neutral Gray**. 모서리는 약간 둥글게 처리하여 '불완전함'을 암시합니다.
*   **라벨링:** [데이터 A], [관련 법규 X], [임상 결과 Y] 등.

#### B. [GAP] 정보 단절 아크 (The $F_{Gap}$ Arc)
*   **기능:** 노드 간의 연결이 끊어지거나, 필수적인 맥락(Context)이 누락된 지점을 시각적으로 정의합니다. **가장 중요한 경고 요소.**
*   **디자인:** 붉은색으로 처리된 곡선 또는 아크(Arc). 이 아크는 단순히 빈 공간이 아니라 '잠재적 위험' 자체를 의미하며, 주변 노드들을 불안하게 감싸는 형태로 표현됩니다.
*   **라벨링 예시:** $\rightarrow$ **[!] 정보 단절 (Missing Edge)**: 자금 흐름 목적성 검증 프로세스 누락 $\rightarrow$

#### C. [PROCESS] EPHA 개입 모듈 (The Edge Module)
*   **기능:** EPHA의 핵심 가치인 '정보 단절 해소 프로세스'가 작동하는 영역을 시각화합니다. 이 단계에서 빛나는 연결(Edge)이 발생합니다.
*   **디자인:** 중앙에 위치하며, **Accent Blue** 계열의 점진적인 그라데이션 배경으로 처리된 모듈 박스. 내부에는 3~5단계의 명확한 순서가 있는 플로우차트 요소를 포함합니다 (Step 1 $\rightarrow$ Step 2...).
*   **핵심 시각화:** 파란색 빛의 흐름(빛나는 Edge)이 Gap Arc를 가로지르며 다음 노드를 활성화하는 애니메이션 콘셉트를 염두에 두고 디자인합니다.

#### D. [END] 해결/통찰 노드 (The Solution Node)
*   **기능:** 프로세스 적용 후, 문제가 해소되고 정량적 통찰(Insight)을 얻는 최종 결과물입니다.
*   **디자인:** 가장 크고 명확하며, **Success Green** 계열로 채워진 사각형 또는 다이아몬드 형태. 배경에 체크 마크($\checkmark$)를 배치하여 완료된 느낌을 극대화합니다.
*   **라벨링 예시:** **[✅ Mitigated Value]**: $F_{Gap}$ 제거에 따른 예상 손실액 회복 ($X$원).

### 3. 📝 아티클 템플릿 적용 가이드라인 (Article Flow Template)

모든 교육 콘텐츠는 다음의 4단계 흐름을 반드시 따라야 합니다.

| 단계 | 목표/제목 | 시각적 요소 배치 | 핵심 컬러 코드 |
| :---: | :---: | :---: | :---: |
| **1. 문제 제기 (Pain)** | "우리는 무엇을 놓쳤는가?" (Gap Identification) | [START] 노드 2~3개 나열 $\rightarrow$ [GAP] 아크로 연결. (혼란스러운 데이터의 시각화) | Red/Gray |
| **2. 원인 분석 (Analysis)** | "이 간극($F_{Gap}$)은 어디서 발생했는가?" | $F_{Gap}$를 계산하는 공식(Formula Block) 제시. (전문성 강화) | Neutral Gray + Formula Font |
| **3. 해결 프로세스 (Process)** | "EPHA의 Edge로 어떻게 연결할 것인가?" | [PROCESS] EPHA 개입 모듈을 중심으로, 단계별 플로우 차트 배치. (가장 큰 비중) | Accent Blue |
| **4. 결론 및 가치 증명 (Relief)** | "결과적으로 무엇이 얻어지는가?" | 최종 결과물 [END] 노드 배치 $\rightarrow$ 정량화된 성공 수치를 명시. | Green + Bold Type |
