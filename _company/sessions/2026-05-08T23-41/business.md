## 💾 $L_{Total}$ 위험 진단 데이터베이스 스키마 명세서 (V1.2)
(개발팀 즉시 활용 가능, 관계형 DB 기반 설계)

본 스키마는 '정보 단절'로 인한 모든 잠재적 손실을 측정 가능한 재무 지표($L_{Total}$)로 전환하기 위한 마스터 데이터 입력 구조입니다. 단순히 데이터를 수집하는 것을 넘어, **측정 단위와 가중치(Weighting Factor)**를 통해 정량적인 위험 점수 계산이 가능하도록 설계되었습니다.

---

### 1. Master Table: `Scenario_Input` (상황별 리스크 발생 기록)
*   **설명:** 사용자가 경험하거나 발견한 특정 비효율성/리스크 시나리오의 기본 정보를 담습니다. 모든 $L_{Total}$ 계산은 이 테이블을 기반으로 시작합니다.

| 필드명 | 데이터 타입 | 필수 여부 | 설명 및 제약 조건 (Constraint) |
| :--- | :--- | :--- | :--- |
| `scenario_id` | INT (PK) | O | 고유 식별자 (Auto Increment) |
| `user_input_date` | DATETIME | O | 리스크 발견 일시. (타임스탬프 기록 목적) |
| `risk_area` | VARCHAR(100) | O | **[필수 입력 1]** 리스크 발생 분야 (FK: `Tbl_RiskArea`) 예: 공급망, IT 시스템, 조직 커뮤니케이션. |
| `silo_point_detail` | TEXT | O | **[필수 입력 2]** 정보 단절이 발생한 구체적 지점 설명. 사용자 서사 기반의 상세 텍스트 (최대 500자). |
| `loss_description` | VARCHAR(255) | O | 이로 인해 발생했다고 추정되는 핵심 손실 내용 요약. |
| `is_premium_input` | BOOLEAN | X | 이 시나리오가 유료 진단 플랜에서만 입력 가능한 고급 리스크인지 여부 (→ **수익화 Hook**). |

### 2. Linking Table: `Tbl_RiskArea` (리스크 발생 분야 마스터)
*   **설명:** $L_{Total}$ 분석의 큰 범주를 정의하며, 이 분류 자체가 서비스의 전문성을 높이는 요소가 됩니다.
*   **예시 데이터 및 가중치 설계:**

| 리스크 영역 코드 | 이름 | 설명 | 기본 가중치 계수 ($\alpha$) |
| :--- | :--- | :--- | :--- |
| AREA_SUPPLY | 공급망 관리 (SCM) | 원자재/부품의 흐름 단절. | 1.5 (높음: 비용 손실 비중 높음) |
| AREA_ITSYS | IT 시스템 연동 실패 | 데이터 파편화, 레거시 시스템 문제. | 2.0 (매우 높음: 시간 및 복구 비용 폭증) |
| AREA_COMMUNICATION | 조직 커뮤니케이션 마찰 | 부서/팀 간 정보 비공유(Silo). | 1.0 (표준: 인적 자원 손실 주도) |
| AREA_MARKET | 시장 예측 실패 | 외부 환경 변화에 대한 대응 지연. | 2.5 (최고: 기회 비용($L_{Opportunity}$) 발생) |

### 3. Core Calculation Table: `Tbl_LossMetrics` (손실 측정 항목 및 계산 로직)
*   **설명:** 리스크 시나리오(`Scenario_Input`)에 의해 발생하는 구체적인 손실 유형을 정의하고, $L_{Total}$ 계산의 기초가 됩니다.

| 필드명 | 데이터 타입 | 필수 여부 | 설명 및 제약 조건 (Constraint) |
| :--- | :--- | :--- | :--- |
| `loss_metric_id` | INT (PK) | O | 손실 지표 고유 ID |
| `scenario_id` | INT (FK) | O | 연결된 시나리오 (`Scenario_Input`)의 ID. |
| `loss_type` | ENUM | O | **[필수 입력 3]** 핵심 자원 손실 유형: {TIME, COST, MANPOWER, OPPORTUNITY} |
| `measurement_unit` | VARCHAR(50) | O | 측정 단위 정의 (예: 시간/주, 원/건, 명일/년). |
| `base_value` | DECIMAL(18, 4) | O | **[측정 값]** 실제 관찰된 손실량. 사용자 입력 또는 시스템 추정값. |
| `weighting_factor` | DECIMAL(5, 2) | O | **[필수 입력 4]** 해당 단위가 가지는 비즈니스 중요도 가중치 (1.0 ~ 3.0). 이 값은 `Tbl_RiskArea`와 연동될 수 있음. |
| `calc_value` | DECIMAL(18, 2) | 자동 | **자동 계산 필드:** $\text{Base\_Value} \times \text{Weighting\_Factor}$. (실제 기여 손실액). |

### 4. $L_{Total}$ 최종 산출 로직 (개발팀 참고용)
$L_{Total}$은 단일 지표가 아닌, 구조화된 점수(Score)로 제시되어야 합니다.

$$
L_{Total} = \sum_{\text{All Silo Points}} (\text{Area}_{\text{Weighting}} \times \text{Metric}_{\text{Calc\_Value}})
$$

*   **$L_{Total}$ 해석:** 총 손실 비용(Cost)을 나타내는 재무적 위기감 지표로 사용자에게 제시합니다. (예: "정보 단절로 인한 잠재적 재무 리스크: 45,300원")
*   **KPI 연동:** $L_{Total}$의 변화율($\Delta L_{Total}$)을 핵심 KPI로 사용하며, 이 수치가 높다는 것은 곧 '진단이 시급하다'는 서사적 근거가 됩니다.
