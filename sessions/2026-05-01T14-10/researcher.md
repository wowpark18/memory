## 📘 1. 필수 지식 파편 (Knowledge Fragments) 유형 정의

단순한 코드(Code Snippet)를 넘어, **의도와 맥락을 담은 3가지 차원의 데이터**가 필요합니다.

| No. | 지식 파편 유형 | 구성 요소 및 설명 | 필요한 핵심 정보 (Why it's needed) |
| :---: | :--- | :--- | :--- |
| **1** | **코드 스니펫 (The What)** | 기능 단위의 실제 코드 조각. 함수, 클래스, 주요 로직 블록 등. (기술적 근거) | 코드가 *어떤 동작*을 하는지(기능 정의). |
| **2** | **관계성 노트 (The How & Why)** | 해당 코드 스니펫이 왜 존재하는지에 대한 설명, 의존성 매핑, 호출 경로 추적 결과 등. (맥락 및 연결점) | 코드가 *어떻게 작동하며*, 다른 시스템/지식과 *어떤 관계*를 맺는지(정보의 흐름). **가장 중요함.** |
| **3** | **교훈 노트 (The Lesson Learned)** | 과거 개발자가 이 코드 블록을 수정하거나 구현하면서 겪었던 시행착오, 아키텍처적 결정의 배경 및 제약 사항. (경험 자산화) | 코드가 *왜 이렇게 되어있는지*, 미래에 유사한 문제를 만났을 때 *어떻게 해결해야 하는지*(위험 회피/최적화). |

## 📊 2. 지식 파편의 양적 기준 및 목표 볼륨 설정 (Minimum Viable Dataset)

MVP(PoC) 단계에서 시스템 학습과 검증이 가능한 최소한의 데이터셋을 확보하는 것이 목표입니다.

| 구분 | 항목 | 필요 최소량 (Metric) | 비고 (측정 기준) |
| :---: | :--- | :--- | :--- |
| **대상 코드 모듈** | 레거시 시스템 코드 베이스 | 5 ~ 10개 핵심 모듈 | 각 모듈은 독립적이지만, 최소 2개 이상의 외부 의존성을 가져야 함. (연결성 테스트용) |
| **관계성 노트** | 연결점(Dependency Linkage) 보고서 | 모듈당 3개 이상 | A $\to$ B, B $\leftarrow$ C 등 방향성이 명시된 관계 데이터가 필수적입니다. |
| **교훈 노트** | Lessons Learned 사례 | 최소 10개 항목 | 'A 기능을 구현할 때 X라는 오류를 발견했으나, Y 방식으로 해결했다.'와 같은 구체적인 서사 구조 필요. |
| **데이터 다양성** | 기술 스택 및 도메인 | 최소 2가지 이상의 이종 조합 | (예: Python 백엔드 + React 프론트)처럼 다양한 기술적 배경의 코드를 포함해야 범용성이 확보됩니다. |

## ⚙️ 3. 지식 데이터 파이프라인 초기 구축 계획 (Data Pipeline Strategy)

수집된 비정형 데이터를 구조화하고 AI가 학습할 수 있는 형태로 표준화하는 2단계 프로세스를 설계합니다.

### Phase 1: Raw Data Collection & Pre-processing (데이터 수집 및 정제 단계)
*   **소스 확보:** 기존 개발 문서, GitHub Commit 메시지(특히 대규모 Refactoring 커밋), 코드 베이스 자체를 주요 소스로 지정합니다.
*   **자동화 도구 활용:** 단순 키워드 검색을 넘어, **정적 분석기(Static Analysis Tools)**와 **코드 커버리지 툴**의 출력을 가져와 '사용되지 않는 코드', '미문서화된 함수' 등 의심 지점을 자동으로 식별합니다.
*   **결과물:** 구조화되지 않은 원본 데이터셋 (Raw Code, Commit Log, Wiki Page).

### Phase 2: Knowledge Extraction & Standardization (지식 추출 및 표준화 단계)
*   **핵심 로직 (AI Core):** 수집된 데이터를 '개념어 $\to$ 관계성 $\to$ 경험'의 구조로 변환하는 것이 목표입니다.
    1.  **Concept Extractor:** 코드 스니펫을 입력받아 핵심 개념(예: `UserAuthentication`, `PaymentGateway`)을 추출합니다. (NLP/LLM 활용)
    2.  **Relation Mapper:** Concept A와 Code B가 만나는 지점을 찾아 관계성 노트의 뼈대를 만듭니다. (`A`는 `B`를 통해 `C`에 영향을 준다.)
    3.  **Narrative Generator:** 발견된 관계성과 코드 흐름을 기반으로, 담당 개발자(혹은 AI)가 "이전 경험에서 배운 교훈" 형식의 보고서 초안을 생성합니다.
