**[핵심 원칙]:** 모든 화면은 사용자가 **문제 해결 과정(Process)**을 눈으로 추적하게 설계되어야 함.

## 🖼️ Wireframe Set Overview

본 PoC 데모는 총 3단계의 흐름으로 구성되며, 특히 Step 2에 EPHA의 핵심 가치인 지식 연결 프로세스가 가장 명확하게 시각화됩니다.

### **[Wireframe 1] 진입 및 문제 정의 (The Input Capture)**
*   **목표:** 사용자의 모호하고 복합적인 요구사항(Purpose)을 시스템으로 유입시키는 단계. 기존 앱의 한계점을 보여주며 EPHA를 필요하게 만듦.
*   **레이아웃 특징:** 검색창 대신 '프롬프트 입력' 형태가 핵심.

```markdown
=============================================================
[화면명: 오늘의 목적 (Purpose Input)]
-------------------------------------------------------------
// [헤더 영역]
[EPHA 로고] | 오늘, 무엇을 위한 하루인가요? (AI 기반 지능형 컨설팅)
-------------------------------------------------------------
// 💡 핵심 기능 영역: Context Capture Zone
[텍스트 입력창 - 와이드]
👉 "강남의 갤러리 전시 관람 후, 격식 있는 저녁 식사" (Placeholder)

[선택적 정보 추가 버튼들]
[📅 일정/시간] [🌡️ 날씨 조건] [🎨 분위기 키워드] [👤 체형/스타일 선호도]

-------------------------------------------------------------
// ⚙️ 하단 안내 및 프로세스 예고
EPHA는 입력된 문맥(Context)을 분석하여, 단순한 데이터 조합이 아닌 '최적의 경험 경로'를 도출합니다.
[버튼: 연결하고 탐색하기 (Search & Map)]

=============================================================
```
**💡 주석 및 설명:**
*   **Pain Point 시각화:** 일반적인 앱은 날짜나 옷장 같은 단일 데이터만 다루지만, EPHA는 '문장(Context)' 전체를 입력받아 문제의 복잡성을 정의합니다.
*   **Adapter Layer 작동 준비:** 사용자의 문장이 곧 Adapter Layer가 분석할 수많은 모듈 간의 연결 키워드가 됩니다.

---

### **[Wireframe 2] 지식 그래프 매핑 프로세스 (The Connection Engine)**
*   **목표:** 입력된 정보(Purpose)가 어떻게 '지능형 시스템'에 의해 구조화되고 관계성을 갖게 되는지를 보여주는 화면. 이 페이지가 EPHA의 핵심 기술력을 증명합니다.
*   **레이아웃 특징:** 좌우 분할 및 중앙 프로세스 시각화 (Knowledge Graph).

```markdown
=============================================================
[화면명: 지식 연결 매핑 중... (Mapping Process)]
-------------------------------------------------------------
// [상단 상태 바]
🔍 분석 중: "갤러리 관람 + 저녁 식사" | 남은 시간: 3초
-------------------------------------------------------------

|=============================================|
|           ✨ 중앙 지식 연결 그래프 ✨         |
|=============================================|
| [Node A: 목적지 분위기 (Gallery)] ---(Connection)--- [Node C: 적정 복장 계열] |
| ^                                            v                                |
| [Node B: 시간대/활동성 (오후-저녁)] <--(Adapter Layer)--> [Node D: 체온 변화 고려 요소] |
|=============================================|

// 🛠️ 프로세스 분석 창 (좌측 패널)
[제목: Context Decomposition Analysis]
1.  **Goal:** 격식 있는 경험 제공 ($\rightarrow$ 핵심 가치 추출)
2.  **Constraint:** 시간적 흐름에 따른 활동 변화(Gallery $\to$ Dinner) ($\rightarrow$ 동선 및 온도 변화 감지)
3.  **Resource Pool:** (옷장 데이터, 날씨 API, 지역 특성 DB 등)

// 🚀 Adapter Layer 작동 표시 (우측 패널 - 핵심 주석)
[✅ ADAPTER LAYER 활성화: 관계성 모듈]
*   **[Source Module: Location/Event Data]** $\to$ **(Adapter)** $\to$ **[Target Module: Fashion Recommendation]**
    *(작동 원리: '갤러리'라는 개념을 '실내 고급스러움', '활동적인 이동'이라는 파라미터로 변환하여 옷장 데이터를 재필터링)*

=============================================================
```
**💡 주석 및 설명:**
*   **지식 그래프(KG):** 단순한 플로우가 아니라, **노드(Node)**들이 **연결선(Connection)**으로 상호작용하는 학술적인 구조를 시각화했습니다. (정보의 연결성 강조)
*   **Adapter Layer:** 이 영역이 핵심입니다. 외부/분산된 데이터 소스(날씨, 장소 정보 등)가 EPHA의 내부 로직에 들어가기 위해 **'번역되고 중재되는 과정'**을 명시적으로 보여줍니다.

### **[Wireframe 3] 구조화된 결과물 (The Aha Moment)**
*   **목표:** 연결 과정을 거쳐 도출된, 단순 추천 이상의 '가이드라인' 형태의 최종 결과를 제시. 사용자가 문제 해결의 근거를 납득하게 만듦.
*   **레이아웃 특징:** 핵심 아이템 나열 대신, 구조화된 섹션과 스토리텔링 요소 배치.

```markdown
=============================================================
[화면명: 오늘의 최적 경로 (Optimal Path for You)]
-------------------------------------------------------------
// 🥇 [핵심 추천] Today's Look & Storytelling Flow
*   **Concept:** '세련된 전환(Sophisticated Transition)' - 하루의 흐름에 맞는 코디 스토리.
*   **Key Piece:** A-라인 트렌치코트 (Gallery $\to$ Dinner 모두 적합) [좌측 이미지]
*   **Styling Tip:** 톤 온 톤 베이지 계열로 통일감을 유지하여, 시간 변화에도 연속성을 확보하세요.

-------------------------------------------------------------
// 📊 [왜 이 코디인가?] — EPHA의 논리적 근거 제시 (필수 영역)
[Section Title: The Logic Behind the Recommendation]
*   **1. Context Matching:** '갤러리(실내, 고급)'와 '저녁 식사(격식)'라는 목적에 모두 대응하는 범용성을 확보했습니다.
*   **2. Transition Analysis (Adapter 결과):** 갤러리 $\to$ 레스토랑으로의 이동 시 발생할 수 있는 **온도/분위기 급변**을 고려하여, 아우터의 보온성과 디자인 연속성을 동시에 충족하도록 설계되었습니다.
*   **3. Resource Optimization:** 옷장 내 [재킷-코트] 조합 중 가장 높은 활용도를 가지는 아이템들을 우선순위로 매칭했습니다.

-------------------------------------------------------------
// 🔗 추가 지식 연결 (Knowledge Deep Dive)
[관련 아티클] '도시의 빛과 그림자: 복합적 상황에 대비하는 스타일링 방법' (읽기)
[Graph View]: 이 코디가 성공적인 이유를 그래프로 확인하기 (View Graph)

=============================================================
```
