## 🌟 PoC 와이어프레임 콘셉트 가이드라인 재정립: 'Aha Moment' 시각화 설계안
(Flow Diagram 중심, 단순 노드/엣지 배열 탈피)

본 가이드는 Writer가 제시한 **Pain $\rightarrow$ Relief**의 서사적 흐름을 기술적으로 구현하는 데 초점을 맞춥니다. 사용자가 단순히 지식이 연결되었다는 사실을 보는 것이 아니라, 마치 '막혔던 사고 회로에 전기가 통하는' 듯한 드라마틱하고 직관적인 경험(Aha Moment)을 하게 만들어야 합니다.

### 🖼️ 1. 핵심 시각적 메타포 (Core Visual Metaphor)
| 개념 | 비유/메타포 | 디자인 방향성 | 사용 목적 |
| :--- | :--- | :--- | :--- |
| **정보 단절 (Pain)** | 혼돈의 스모그(Smog), 엉킨 전선 다발, 깨진 유리 조각. | 낮은 채도의 어둡고 무거운 배경색. 불안정한 레이아웃과 중복되는 정보 블록 배치. | 사용자에게 '문제'와 '비효율성'을 감정적으로 각인시키는 도입부. |
| **EPHA의 개입 (Process)** | 지능형 에너지 흐름, 그리드 오버레이(Grid Overlay), 광학 추적선(Lidar Trace). | 중립적이면서도 역동적인 애니메이션 효과와 시안 블루/골드를 활용한 빛나는 경로 강조. | 시스템이 정보를 수집하고 분석하는 과정을 '관찰 가능하게' 보여주는 핵심 단계. |
| **깨달음 (Aha Moment / Relief)** | 명료화된 청사진(Blueprint), 최적화된 길, 구조적인 계단식 흐름. | 높은 채도의 밝은 배경색으로 전환. 정보들이 정렬되고 간결해진 레이아웃. 핵심 경로에만 초점을 맞춘 '선택과 집중'의 미학. | 문제 해결을 통해 얻는 명확한 결론과 실행 가능한 인사이트를 제공하는 최종 단계. |

### 📐 2. 프로세스 기반 와이어프레임 컴포넌트 (3단계 플로우)

#### 💡 Stage 1: Pain Point Visualization (문제 제기 - 혼돈의 시각화)
**목표:** 사용자가 자신의 문제 상황을 '불편함'으로 느끼게 만든다.
*   **컴포넌트명:** **[Context Overload Grid]**
*   **디자인 요소:** 텍스트 카드, 데이터 블록, 이미지들이 무질서하게 오버랩되거나 비정렬된 그리드 형태로 배치됩니다. 각 블록은 출처(Source A, Source B)가 다르고 크기가 제각각입니다.
*   **상호작용 (Interaction):** 화면이 미세한 떨림(Shaking Effect)이나 노이즈 효과를 가지며 불안정하게 느껴지게 합니다. 사용자가 특정 블록을 클릭해도 명확한 답 대신 "관련 정보가 너무 많습니다"라는 메시지만 나옵니다.
*   **색상 코드:** Background: `#1A202C` (딥 네이비), Highlight: `None` (무채색의 불안정함).

#### ⚡ Stage 2: EPHA Process Visualization (개입 - 흐름 추적)
**목표:** 시스템이 '지식 연결'이라는 행위를 눈에 보이게 만든다.
*   **컴포넌트명:** **[Adaptive Flow Tracer]**
*   **디자인 요소:** 화면 중앙에서 시작된 질문(Query)을 중심으로, 주변의 무작위 정보 블록들(Stage 1 잔여물)로부터 빛나는 '엣지(Edge)'가 점차적으로 모입니다. 이 엣지는 단순한 선이 아니라, **'관계성 가중치(Relationship Weight)'**에 따라 두께와 밝기가 변하는 에너지 라인처럼 보입니다.
*   **상호작용 (Interaction):** 에이전트/시스템의 분석 과정을 시뮬레이션합니다. 특정 지식 노드(Node)가 활성화될 때마다, 해당 노드의 **메타데이터**가 팝업으로 뜨며 "출처: 레거시 문서 A, 관련 기술: Python Async,"와 같이 연결 이유를 설명합니다 (정보 단절 해소 과정 시각화).
*   **색상 코드:** Background: `#1A202C` $\rightarrow$ Gradient Transition. Flow Line: `rgba(59, 130, 246, 0.8)` (시안 블루), High Weight Link: `rgba(252, 211, 77, 0.9)` (골드).

#### ✨ Stage 3: Aha Moment Visualization (해결 - 명료화)
**목표:** 혼돈 속에서 최적의 답이 발견되는 '깨달음'을 극대화한다.
*   **컴포넌트명:** **[Optimized Blueprint View]**
*   **디자인 요소:** 모든 엣지와 노드가 사라지거나 축소되고, 오직 문제 해결에 가장 중요한 *최적의 경로(Optimal Path)*와 *핵심 원리(Root Cause)*만 남습니다. 이 경로는 마치 청사진 위에 그려진 듯한 깨끗하고 구조적인 라인으로 표현됩니다 (Blueprint Style).
*   **상호작용 (Interaction):** 'Aha!' 순간을 나타내는 애니메이션이 필수입니다. 배경의 노이즈가 걷히면서(De-Clutter Effect) 화면 전체가 밝고 명료한 상태로 전환되고, 최종 결과물은 **'Key Takeaways'**와 **'Next Action Steps (Actionable Items)'** 섹션으로 구조화됩니다.
*   **색상 코드:** Background: `#F7FAFC` (밝은 그레이), Primary Accent: `indigo-600` (강한 신뢰감).
