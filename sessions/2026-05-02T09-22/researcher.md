### Ⅰ. 예측 알고리즘 필수 핵심 데이터 모델 및 변수 (Variables & Data Models)

펫/헬스케어 분야에서 '정상 범위 이탈'을 감지하는 것이 핵심이므로, 단일 측정치보다는 **시간의 흐름에 따른 변화(Temporal Change)**를 포착하는 시계열 데이터를 중심으로 모델링해야 합니다.

#### 1. 필수 데이터 카테고리 및 변수 (Variables)
| 카테고리 | 상세 변수 (Variable) | 측정 단위/형식 | 예측 가치 | 활용 기술 |
| :--- | :--- | :--- | :--- | :--- |
| **활동성** | 🚶‍♀️ 평균 이동 거리 (Daily Step Count) | m / 시간당 변화율 (%) | 활동량 급감: 통증, 질병 초기 신호 감지 | Time-Series Analysis |
| | 💤 수면 패턴 변이 (Sleep Cycle Metrics) | 분/주기(REM/Deep Sleep 비율) | 불규칙성 증가: 불안정, 스트레스 또는 신경계 이상 예측 | Pattern Recognition |
| **생체 지표** | ❤️ 심박수 및 HRV (Heart Rate & HRV) | bpm / SDNN, RMSSD 값 (ms) | 기저치 대비 높은 변이: 급성 질환, 염증 반응 가능성 측정 | Anomaly Detection |
| | 🌡️ 체온 변화 주기 (Temperature Fluctuation) | °C / 평균 및 최고/최저 편차 | 지속적 상승 또는 하강: 감염, 내분비계 이상 예측 | Trend Analysis |
| **환경/행동** | 🍽️ 식사 패턴 변이 (Feeding Pattern Shift) | g/회식 횟수 변화 (%) | 식욕 부진(Anorexia), 소화기 문제의 초기 신호 포착 | Correlation Mapping |
| | 🌐 환경 노출 데이터 (Environmental Data) | CO2 농도, 습도, 미세먼지 지수 (PM2.5) | 환경적 스트레스 요인과의 연관성 분석 | External API Integration |

#### 2. 핵심 성능 측정 지표 (Metrics / Output Scores)
시스템이 사용자에게 제공해야 할 **정량적인 '경고 수준'**입니다. 단순히 변수를 나열하는 것이 아니라, 이를 종합하여 가중치를 부여한 점수화가 필수적입니다.

1.  **🚨 위험 지수 (Risk Index Score):** 0~100점 만점. 현재 수집된 모든 변수가 정상 범위에서 벗어난 정도를 통합 산출합니다. 이 점수를 기반으로 '경고' 등급(Low/Medium/High)을 결정합니다.
2.  **📈 변화율 이상 지수 (Rate-of-Change Anomaly Score):** 특정 변수의 **일주일 평균 대비 급격한 하락 또는 상승 폭**에 초점을 맞춥니다. (예: 평소 활동량이 50% 감소했다는 것 자체가 위험 신호).
3.  **🔮 예측 가능성 점수 (Predictive Probability Score):** 현재의 패턴을 유지할 경우, 특정 질병(예: 관절염 악화)이 발생할 확률을 백분율로 제시합니다.

### Ⅱ. 데이터 수집 및 활용 시 법규/윤리적 가이드라인 (Compliance & Ethics)

건강 관련 데이터는 민감 정보(Sensitive Data)에 해당하므로, 개발 초기 단계부터 최고 수준의 보안과 윤리 기준을 적용해야 합니다.

#### 1. 개인정보 보호 법규 준수 사항
*   **데이터 주권 및 동의 (Consent):** 모든 데이터를 수집하기 전에 **'어떤 데이터가', '왜', '얼마나 오랫동안 사용될지'**에 대한 명확하고 분리된(Granular) 동의를 받아야 합니다. 특히, 진단 목적으로 활용된다는 점을 고지해야 합니다.
*   **데이터 익명화/가명화 (Anonymization/Pseudonymization):** 시스템 내부 처리 및 학습용 데이터는 개인 식별 정보(이름, 정확한 주소 등)를 즉시 제거하거나 대체 키로 변환하여 저장해야 합니다. (GDPR, HIPAA 준수 원칙 적용)
*   **데이터 보관 기간 명시:** 사용자가 언제 자신의 데이터를 삭제할 수 있는지, 그리고 그 데이터가 시스템에서 완전히 파기되는 절차(Right to be Forgotten)를 명확히 제공해야 합니다.

#### 2. AI 윤리적 고려 사항 (Bias & Transparency)
*   **결과 해석의 투명성 (Explainability - XAI):** AI가 "High Risk"라는 진단을 내렸을 때, 단순히 경고 메시지를 주는 것만으로는 부족합니다. **"이런 변수(예: HRV 급락)와 저런 변수(예: 활동량 감소)의 조합 때문에 위험하다고 판단했습니다."** 와 같이 근거를 제시해야 신뢰성을 확보하고 법적 책임을 분산시킬 수 있습니다.
*   **편향성 완화 (Bias Mitigation):** 학습 데이터가 특정 품종, 지역, 경제 수준의 동물에게만 치우쳐서는 안 됩니다. 다양한 인구통계학적 데이터를 포함하여 모델이 모든 사용자 그룹에 공평하게 작동하도록 설계해야 합니다.
