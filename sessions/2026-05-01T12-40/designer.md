### 🔡 2. 타이포그래피 (The Typeface: 가독성과 모던함)

*   **전체 스타일:** 간결하고 기하학적인 느낌을 주는 산세리프(Sans-serif) 계열을 유지하여 '디지털' 느낌을 극대화합니다.
*   **주요 폰트 세트 (Primary Font Set):**
    *   **Headline/Diagram Title:** **Pretendard Bold** (가장 강력하고 구조적인 인상을 주며, 대문자(ALL CAPS)와 결합 시 기술적 느낌이 극대화됨).
    *   **Body Text/Paragraph:** **Noto Sans KR Regular** (가독성이 매우 높고 학술 문서에 적합하여 긴 텍스트를 처리하는 데 안정감을 제공함).
    *   **Code Block/Technical Terminology:** Monospace Font (예: Fira Code, Consolas) - 구조와 코드를 시각적으로 분리.

### 📊 3. 지식 다이어그램 표준 포맷 (The Knowledge Diagram Set)

복잡한 기술적 과정을 설명하는 데 최적화된 3가지 버전의 표준화된 다이어그램을 제안합니다. 모든 다이어그램은 **`Accent Solution (#00C4FF)`**으로 연결선과 핵심 노드의 테두리(Stroke)를 강조하고, 부드러운 글로우 효과를 적용하여 '데이터 흐름'의 느낌을 줍니다.

#### 📐 Diagram Version A: 아키텍처 레이어 모델 (The Adapter Layer Focus)
*   **용도:** 시스템 간의 분리와 중재 메커니즘(예: EPHA와 외부 앱의 관계).
*   **구조:** 수직적 계층 구조(Vertical Stacking)를 채택합니다. 각 컴포넌트는 독립된 박스(Component Box)로 표현됩니다.
*   **시각 요소:**
    1.  **Layer Boxes:** 배경색 `Secondary Structure`를 사용하고, 테두리 두께를 2px로 설정하여 '경계'의 개념을 명확히 합니다. (예: [External App] $\rightarrow$ [Adapter Layer] $\rightarrow$ [Core Logic])
    2.  **Connection Line:** 연결선은 단순 직선이 아닌, **꺾이는 화살표(Bend Arrow)**를 사용하여 데이터가 흐르는 방향성을 강조합니다. 이 선의 색상과 글로우를 `Accent Solution`으로 처리합니다.
    3.  **핵심 주석:** 각 레이어 경계에는 'IP 완전 양도 전제'와 같은 **규칙적 조건 (Constraint Box)**을 작은 박스로 삽입하여 기술적 제약 조건을 시각화합니다.

#### 🔗 Diagram Version B: 구조 지식 그래프 모델 (The Knowledge Graph Focus)
*   **용도:** 비선형적인 개념 간의 관계성, 맥락 연결(예: '정보 파편' $\rightarrow$ '맥락 발견' $\rightarrow$ '해결책 도출').
*   **구조:** 중앙 노드에서 사방으로 뻗어나가는 **유기적 네트워크(Organic Network)** 형태를 취합니다.
*   **시각 요소:**
    1.  **Nodes:** 주요 개념은 크고 둥근 모서리의 원형 박스(`Primary Depth` 배경, `Accent Solution` 테두리)로 표현합니다.
    2.  **Edges (연결선):** 연결선은 단순 화살표가 아닌, **'관계의 힘(Force of Relationship)'**을 상징하는 곡선 형태의 흐름으로 묘사하며, 이 선 위에는 관계를 정의하는 키워드(예: "관련성", "영향")를 작게 삽입합니다.
    3.  **중앙 허브:** 모든 지식이 수렴하거나 출발하는 중심 노드는 가장 크고 입체적인 효과를 주어 'EPHA Core'의 역할을 강조합니다.

#### ⚙️ Diagram Version C: 프로세스 플로우 모델 (The Workflow Focus)
*   **용도:** 시간 순서에 따른 구체적인 작업 흐름이나 단계적 과정(예: 사용자의 문제 정의부터 해결책 도출까지).
*   **구조:** 왼쪽에서 오른쪽으로 전개되는 **선형의 스테이지(Linear Stages)**를 채택합니다.
*   **시각 요소:**
    1.  **Stages:** 각 단계를 세로로 길쭉한 사각형 박스로 정의하며, 단계 번호와 제목을 명확하게 기입합니다. (Phase 1: Problem Definition)
    2.  **Progression Arrow:** 단계 간의 이동은 크고 두꺼운 `Accent Solution` 화살표를 사용하여 **'진행성(Progression)'**을 강조합니다. 이 화살표는 마치 전기가 흐르듯 부드러운 그라데이션 효과가 필요합니다.
    3.  **Decision Point:** 과정 중간에 'Yes/No'와 같은 판단 지점은 마름모꼴(Diamond Shape)의 Decision Box를 사용하여 논리적 흐름을 시각화하고, 이 박스 경계는 `Secondary Structure`로 처리합니다.
